# 1.总则
## 1.1.目的与背景
* 公民币依据《公民宪法》发行，宪法原文：https://github.com/ChinaNation/FRC/blob/main/README.md
* 基于区块链技术以推动“公民建国运动”的主权区块链系统，创立公民币区块链，采取去中心化民运，以建立自由民主的中华民族联邦共和国；
* 公民币区块链是一条不受任何机构掌控的主权区块链，一个所有人都能自由使用的法定数字货币系统，一条所有中华民族联邦共和国公民都能参与治理的区块链；

## 1.2.名称与结构  
* 名称：公民币区块链（英文：CitizenChain），原生数字货币：公民币，符号：GMB；
* 单位：常用单位：元（YUAN），小数单位：分（FEN），元最小为1元，分最小为1分，100分等于1元，统一使用分为系统计算单位，使用元为前端展示单位。

## 1.4.创世理念
* 公民宣言：先有人类后有国家，是公民建立国家，国家是公民的国家，是公民治理国家，而不是国家统治公民，公民没有爱国的义务；国家政权的建立其基本原则是保护公民的生命权、自由权、财产权、反抗压迫权和选举与被选举权不受任何的非法侵犯，当国家政权无法保证这一基本原则时，公民有权有义务推翻这个政权，建立一个以保障公民生命权、自由权、财产权、反抗压迫权和选举与被选举权为基本原则的政权。
* 国家名称与五民主义：中华民族联邦共和国国家名称是基于中华各民族悠久历史与璀璨文化的沉淀，全称为：中华民族联邦共和国，简称为：中华联邦，或中国及中华民国；中华民族联邦共和国是致力于推行五民主义———公民治理国家（民治）、实现民主共和（民主）、保障公民权利（民权）、建设民生社会（民生）、复兴民族文化（民族）———的联邦制共和国。
## 1.5.开源与去中心化
* 公民币区块链及其附属软件采用MIT开源协议，并在GitHub上开放源代码，以构建一套使开发者能快速、简易开发部署区块链应用的系统；
* 公民币区块链坚持去中心化，任何个人、任何组织都可以加入成为去中心化的节点（全节点、轻节点），及参与公民币区块链的治理与技术开发。

# 2.节点设置
## 2.1.节点概览
```
|  节点  |  国储会权威节点  |   初始省储会权威节点  |  初始省储行权益节点  |     全节点     |          轻节点       |
|-------|----------------|--------------------|-------------------|---------------|----------------------|
|  数量  |       1个      |        43个        |        43个        |      无限     |      公民/访客：无限    |
|  功能  |    国家铸币权   |       省铸币权       |    链下交易验证     |     PoW共识   |     链上治理、跨境支付   |
```
![alt text](https://raw.githubusercontent.com/ChinaNation/GMB/main/wenku/节点图.png)

## 2.2.国储会权威节点  
* 有1个国储会权威节点，不能增删改，拥有19个管理员，分别对应国家储备委员会的19个委员；
* 国储会拥有19票投票权和1个多签名治理账户（duoqian_address），采用N=19，T≥13签名，通过则19票同时生效，反之则同时否决；
* 国储会拥有发起“发行数字公民币、销毁数字公民币、状态转换升级、更换国储会多签管理员、增删改新省储会/新省储行”等提案权，承担数据归档和节点引导。

## 2.3.省储会权威节点 
* 初始43个省，每省1个省储会权威节点，初始权威节点不能增删改，每个节点拥有9个管理员，分别对应省储备委员会的9个委员；
* 每个省储会拥有1票投票权和1个多签名治理账户（duoqian_address），采用N=9，T≥6签名，签名通过则生效，反之则否决；
* 省储会拥有发起“销毁数字公民币、更换省储会多签管理员”等提案权，承担数据归档和节点引导。

## 2.4.省储行权益节点    
* 初始43个省，每省1个省储行权益节点，初始权益节点不能增删改，每个节点拥有9个管理员，分别对应省储行的9个董事会成员； 
* 每个省储行拥有1票投票权和1个多签名治理账户（duoqian_address），采用N=9，T≥6签名，签名通过则生效，反之则否决；
* 每个省储行拥有1个无私钥账户（keyless_address）无私钥账户只能接收数字公民币，用于永久质押省储行创立发行的数字公民币；
* 省储行拥有发起“销毁数字公民币、更换链下交易验证密钥、更换省储行多签管理员”等提案权，承担数据归档和节点引导。

## 2.5.全节点    
* 全节点是数据归档和新块铸块节点，仅负责数据归档和所有新区块的铸造，新区块特指除创世区块以外的所有区块，采用PoW共识机制获得铸块权；  
* 全节点数量不限，部署运行citizennode软件（中文：公民币区块链节点软件）的即全节点，任何组织、任何人均可下载安装节点软件成为全节点。

## 2.6.轻节点：公民/访客 
* 使用WuminApp（公民币）轻节点软件，使用身份识别码（SFID）绑定的即公民轻节点，SFID和公民轻节点任意公钥地址一对一绑定，完成绑定后可获得投票权，一个SFID只能同时与一个公钥地址绑定。  
* 安装WuminApp轻节点软件未绑定的,为访客轻节点，访客轻节点没有投票权，使用第三方钱包的，亦为访客轻节点。

# 5.自定义模块/pallets
## 5.1.治理模块/governance
### 5.1.1.投票系统/voting-engine-system
* 治理投票仅分为三种类型：内部投票、联合投票和公民投票，且投票选项只有同意或反对两个选项，每种投票的最长时限均为“30天”，超时没有通过的则否决，投票统一使用区块高度计数；
* 内部投票仅在国储会、各省储会、各省储行内部进行投票，且内部投票仅适用于“内部事项”，内部投票通过则执行，超期或未通过则否决；
* 联合投票是指国储会+全体省储会+全体省储行之间的联合投票，除由内部投票的事项外，其他所有需要投票的事项均是联合投票事项，联合投票结果为全票通过的则直接执行，非全票通过的或超期的则进入公民投票流程；
* 公民投票是指所有经过公民认证的轻节点均可参与的投票，超过50%的可投票轻节点同意的则通过，超期或未通过的则否决；
* 国储会节点有19票，省储会节点每省1票共43票，省储行节点每省1票共43票，轻节点通过绑定身份识别码的拥有1票投票权；
* 投票资格由身份识别码系统判定，区块链仅负责提案期内一次性计票与结果不可篡改。

  ![alt text](https://raw.githubusercontent.com/ChinaNation/GMB/main/wenku/联储会架构图.png)
* 投票分类：
   * 内部投票/internal_vote.rs ：在国储会、各省储会、各省储行内部进行投票，投票阈值详见：primitives/src/count_const.rs常量文件；
   * 联合投票/joint_vote.rs    ：联合投票直接记录各机构（国储会+各省储会+各省储行）内部投票结果，按以下“联合投票流程图”执行；
   * 公民投票/citizen_vote.rs  ：使用SFID绑定的账户地址可参与投票，未绑定的不能投票，超过50%的可投票轻节点同意的则执行，反之或超时则否决；
* 联合投票流程图：
```
① 国储会任意管理员发起联合投票提案
② 各机构参与投票：国储会19票+43个省储会43票+43个省储行43票=105票
③ 判断结果：
   ├─ 全票通过 → 直接执行
   └─ 非全票/超期 → 进入公民投票流程
         ├─ >50% → 执行
         ├─ ≤50% → 否决
         └─ 超期  → 否决
```

### 5.1.2.状态升级/runtime-root-upgrade
* 状态转换函数升级/runtime_upgrade.rs
* 链的运行时状态转换函数升级（runtimee升级）由国储会任意管理员发起提案，经联合投票决定，通过则升级，反之则否决。

### 5.1.3.管理员权限治理/admins-origin-gov
* 更改国储会、省储会、省储行管理员列表成员的投票，本投票为内部投票，仅能对管理员列表进行更换，不能增加或删除，
* 国储会拥有19个管理员，每个省储会和省储行各拥有9个管理员；
* 国储会仅能发起更换国储会管理员的内部投票，各省储会/各省储行仅能发起更换本省管理员的内部投票；

### 5.1.4.决议发行治理/resolution-issuance-gov
* 仅国储会可提案决议发行，本提案为联合投票提案；

### 5.1.5.决议销毁治理/resolution-destro-gov
* 国储会可提案销毁所持有治理账户内的公民币，由任意国储会委员/管理员提案，本提案为内部投票提案；
* 省储会可提案销毁所持有治理账户内的公民币，由任意省储会委员/管理员提案，本提案为内部投票提案；
* 省储行可提案销毁所持有治理账户内的公民币，由任意省储行懂事/管理员提案，本提案为内部投票提案。

## 5.2.发行模块/issuance
### 5.2.1.省储行质押利息/shengbank-stake-interest
* 初始省储行节点，每个省储行发行该省总人口数x100的数字公民币，各省人口以中共第7次人口普查数为准（并做省份调整），共计发行144,349,737,800元；
* 各省储行创立发行的数字公民币永久质押于该省储行质押地址（keyless_address），该地址为无私钥地址，永久质押；
* 各省储行质押的数字公民币，由区块链发行质押利息,质押利息存入各省储行治理账户（duoqian_address），利息归各省储行所有，用于省储行运营和资助民运人士；
* 质押利息初始年利率为1%，并以线性衰减的方式每年减少0.01%，100年后停止计算利息，共产生利息72,896,617,589.00元公民币；  
* 省储行keyless质押地址由每个省的总人口数，通过Blake3哈希算法生成，各省总人口数详见：primitives/china/china_ch.rs/citizens_number；
* 每年=87600个区块，由pow_const.rs常量中的区块与时间参数定义，即每87600个区块执行一次利息发放及利率衰减，共执行100次后永久停止。

### 5.2.2.全节点发行/fullnode-pow-reward
* 运行全节点的每铸造一个新区块，系统发行9999.00元数字公民币用于奖励该节点，全节点铸块发行为第1个至第9,999,999个区块，共计发行量为99,989,990,001.00元数字公民币；
* 当区块高度超过9,999,999个区块后（即第10,000,000个起，含本数），即永久停止全节点发行，全节点再铸造新块的将不再获得铸块奖励;
* 全节点发行模块仅负责发行，不参与铸块，铸块用Substrate框架自带的PoW共识，全节点通过PoW工作量证明获得铸块权后，由全节点发行模块发行公民币予以奖励。

### 5.2.3.公民轻节点发行/citizen-lightnode-issuance
* 使用身份识别码（SFID）完成公民轻节点认证的，将获得认证奖励，可获得认证奖励的节点总量为1,443,497,378个，前14,436,417个认证的公民轻节点，每个认证的节点奖励9999.00元；第14,436,417个之后再认证的公民轻节点，每个认证的节点奖励999.00元；   
* 达到公民轻节点发行总量后，再认证的节点不再获得奖励，认证发行奖励以先完成认证优先获得，每个身份识别码仅能获得一次认证奖励；  

### 5.2.4.决议发行/resolution-issuance-iss
* 成立联邦公民储备委员会后，由联储会决议发行数字公民币，经联储会联合会议决议通过后，由国储会权威节点提案发起发行；  
* 决议发行执行模块仅负责发行执行，提案、投票由决议发行治理模块（governance/resolution-issuance-gov）负责，治理模块通过后，发行模块执行发行；
* 发行模块根据决议发行提案铸造新公民币，所铸造的新币由各省储会多签名治理账户（duoqian_address）持有。

## 5.3.交易模块/transaction
### 5.3.1.链上交易/onchain-transaction-pow
* 链上交易手续费为0.1%，精度为四舍五入后的小数点后两位，且单笔支付手续费最少为0.1元，不足0.1元的以0.1元计算，由付款方必须支付，交易费按8:1:1的比例分账；  
* 链上交易费的80%用于奖励铸块的全节点，10%支付给国储会治理账户（0x34920e9a26bd6338ba9108247d2b2137dc7a6942e724b8673cfa424c82fd9635）用于社区建设和国储会节点运营，10%直接销毁，用于代币通缩燃烧；  
* 链下交易上链的，除省储行权益节点获得的手续费外，其链下交易的金额不再支付链上交易费，例：小李支付1000元公民币给小习，由广东省储行节点验证，其费率为0.1%，那么小李需要支付1001元，小习收到1000元，广东省储行节点收到1元，打包该交易上链时，那1000元不用支付链上交易费，而广东省储行获得的1元需要支付链上交易手续费。 

### 5.3.2.链下交易/offchain-transaction-pos
* 链下交易手续费由各省储行节点自行设定，可设定的范围为0.01%至0.1%，精度为四舍五入后的小数点后两位，且单笔支付手续费最少为0.01元，不足0.01元的以0.01元计算，链下交易手续费由执行清算的省储行节点所有；  
* 链下交易手续费由付款用户承担，根据收款方所使用的省储行清算费率在付款时同时支付，收款用户每年可更换一次清算省储行；
* 链下交易手续费完全由负责清算的省储行节点所有，收入到各省储行多签名治理账户（primitives/src/shengbank_nodes_const.rs/duoqian_address）；
* 各省储行打包上链采用：条件A：交易笔数 ≥ N or 条件B：距离上次打包 ≥ T 双阈值触发打包，N = 100,000 笔，T = 60 分钟，阈值以优先触发为准。
