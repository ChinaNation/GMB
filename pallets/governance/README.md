# 1.新增省节点/new-sheng-node
## 1.1.功能需求
* 仅国储会节点可提案增删改新省储会/新省储行，且必须两者同时增删改，即同一个提案增删改一对省储会/省储行；
* 提案在现有国储会+省储会+省储行之间投票，全票通过的立即执行，非全票通过的则进入“公投流程”，超过可投票节点总数50%投票通过的则执行，反之则驳回；
* 可投票节点仅指现有国储会节点、省储会节点、省储行节点+公民轻节点，且公民轻节点需要验证其CIIC码是否有效，是否对同一提案重复投票等；

# 2.决议销毁治理/resolution-destro-gov
# 2.1.功能需求
* 国储会/省储会均可提案销毁所持有地址内的公民币，国储会的销毁提案仅在国储会内部投票，仅由任意国储会委员/管理员提案，使用门限签名：N=19，T≥13即通过，反之则否决；
* 省储会的销毁提案仅在省储会内部投票，仅由任意省储会委员/管理员提案，使用门限签名：N=9，T≥6即通过，反之则否决；
## 2.2.技术开发

# 3.决议发行治理/resolution-issuance-gov
# 3.1.功能需求
* 仅国储会可提案决议发行，所有决议发行提案需在国储会+省储会+省储行之间投票，全票通过的立即执行，非全票通过的则进入“公投流程”，超过可投票节点总数50%投票通过的则执行，反之则驳回；
* 可投票节点仅指现有国储会节点、省储会节点、省储行节点+公民轻节点，且公民轻节点需要验证其CIIC码是否有效，是否对同一提案重复投票等；
* 决议发行治理模块仅负责提案发行、投票流程，不负责执行发行，投票通过后的发行任务由决议发行发行模块（issuance/resolution-issuance-iss）执行；
