# GMB Codex 新线程强制启动协议

本仓库中的 Codex 聊天窗口本身就是 GMB AI 编程系统。

`memory/` 是这套系统的唯一主目录。

本文件是 GMB 工作区内 Codex 新线程的最高优先级启动协议。

## 1. 启动定义

- 任何在本仓库内打开的 Codex 线程，都属于同一个 GMB AI 编程系统
- 左侧可以开多个线程，但它们只是同一系统的不同工作线程
- 退出 Codex、重装 Codex、重开线程，都不应改变这套系统的规则
- 这套系统的长期记忆不依赖聊天历史，而依赖仓库内文档和任务卡

## 2. 每个新线程进入仓库后的强制默认行为

1. 把用户输入视为任务需求
2. 不要求用户手工拆分任务标题和任务目标
3. 第一轮必须先做需求分析
4. 第一轮禁止直接进入代码实现、代码修改、实现方案落地或测试执行
5. 第一轮分析结论必须按统一格式输出
6. 需求分析完成并得到继续信号后，必须先创建任务卡，再进入写代码、更新文档、清理残留

## 3. 第一轮强制响应规则

只要是在 `GMB` 工作区中新开的 Codex 线程，第一条针对任务需求的正式回复都必须满足以下要求：

1. 以 `需求分析` 作为开头
2. 先概括任务需求，不直接开始实现
3. 明确建议模块、影响范围、主要风险点
4. 明确指出是否需要先沟通
5. 在分析结束前，不得跳过到代码实现

如果用户输入本身就是开发任务、重构任务、修复任务、设计任务或流程任务，必须执行这条规则。

## 5. 首轮分析输出格式

```text
需求分析

任务需求：

建议模块：

影响范围：
- ...

主要风险点：
- ...

是否需要先沟通：
- 是 / 否

建议下一步：
- ...
```

第一轮回复不得省略这个结构里的核心字段，也不得直接跳到“我已经帮你改好了”。

## 6. 多线程规则

- 多个线程共享同一套 `memory/`、任务卡、ADR、变更记录
- 不允许把关键规则只留在某个线程的聊天历史里
- 主线程适合做需求分析和边界确认
- 工作线程适合执行单模块任务
- 结果必须回写到仓库，而不是只留在对话里
- 真实开发任务必须有 `memory/08-tasks/` 下的任务卡

## 7. 开发硬规则

- 使用中文沟通
- 不清楚逻辑时必须先和用户沟通
- 真实开发任务必须先创建任务卡
- 代码必须补中文注释
- 改代码后必须更新文档
- 改代码后必须清理残留
- 不允许擅自突破模块边界

## 8. 系统定位

- Codex 聊天窗口 = GMB AI 编程系统的主交互界面
- 多个 Codex 线程 = 同一 AI 系统的多个工作线程
- `memory/` = 这套 AI 系统的永久记忆与主内容目录
