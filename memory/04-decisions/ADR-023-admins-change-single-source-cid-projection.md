# ADR-023：管理员钱包集合、机构岗位任职与 CID 投影真源

状态：Accepted（2026-06-21；2026-07-13 按机构/岗位/管理员模型整体修订）。

## 决策

管理员不是独立身份类型，也不拥有第二套管理员 CID。机构管理员由以下事实共同成立：

```text
机构 + 管理员钱包账户 + 有效机构岗位任职
```

- `public-admins`、`private-admins` 只保存机构当前有效管理员钱包集合 `admins` 及账户状态。
- 机构信息、岗位定义和岗位任职唯一真源在对应 `entity` pallet。
- entity 接收创世或治理结果后，在同一事务中更新岗位/任职，并从全部有效任职去重派生 `admins`；业务端不得直接改机构管理员集合。
- 个人多签完全独立，由 `personal-admins` 保存账户和管理员集合，并保留自己的 `propose_admin_set_change`。
- OnChina PostgreSQL 只保存登录、节点绑定和公开机构目录投影，不是管理员资格、岗位或权限真源。
- 具体业务权限由业务模块按“机构 + 有效岗位 + 业务动作”硬规则判定，不建立通用岗位权限字符串或权限表。

## 机构创建与后续变更

- 注册局创建机构时提交岗位定义 `roles` 和初始任职 `assignments`；机构管理员钱包集合由 entity 派生。
- 任职来源只允许 `Genesis / Registry / PopularElection / MutualElection / NominationAppointment`，并由 `assignment_source_ref` 指向制度记录。
- 后续换届、任免或岗位变化只能由对应业务模块调用治理底座，或由投票引擎终态结果回调；业务模块不得实现投票流程，投票引擎不得保存岗位或管理员副本。
- 五类固定创世机构岗位定义和席位受治理骨架保护；管理员钱包可依法轮换，但必须持续满足固定席位结构。

## 联邦注册局

- FRG 只有一个机构管理员账户记录，`PublicAdmins::AdminAccounts` 保存 215 个钱包账户。
- 43 省 × 5 席的省域归属由 `PublicManage::InstitutionRoleAssignments` 中稳定岗位代码 `PROVINCE_COMMISSIONER_<省码>` 表达。
- 虚拟省组、本地管理员播种、省组权限表和虚拟管理员账户均已废弃，不得恢复。
- OnChina 登录先匹配 FRG `admins` 钱包集合，再按该钱包的有效省专员任职确定省域；链读失败必须关闭失败。

## 安全边界

- 普通读取可以使用短期本地投影；登录和高危动作提交前必须实时复查链上 `admins` 与有效岗位任职。
- 任职失效、任期过期或钱包移出派生管理员集合后，不再具有对应机构权限。
- 机构法定代表人是机构公开信息，不从管理员集合推导，也不要求等于任何管理员。
- 不保留旧机构管理员资料结构、旧机构管理员集合变更 call、旧虚拟省组或兼容分支。

## 重新创世与验收顺序

1. 重生 chain spec，写入机构信息、五类固定岗位、固定任职和纯钱包管理员集合。
2. 启动节点并通过 Node Guard，核验岗位、席位、任职与管理员派生集合一致。
3. OnChina 同步机构公开投影，联合链读管理员钱包和 entity 任职。
4. 使用真实 PostgreSQL、HTTP、页面和冷签载荷完成全链路验收。
