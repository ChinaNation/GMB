# ADR-015 账户级内部投票管理员模型

- 状态：Superseded
- 原决议日期：2026-05-07
- 被替代：ADR-023 与任务卡 `20260628-institution-admin-field-model-onchain.md`

## 替代原因

原决议把机构管理员错误建模为可直接变更的账户级管理员集合，并允许机构业务直接写管理员真源。2026-07-12 起，机构管理员、机构岗位和任职关系已重新收口，原机构部分不再有效，也不保留兼容路径。

## 当前有效边界

- 机构岗位和机构管理员任职唯一真源归 `entity`。
- `public-admins` / `private-admins` 只保存由有效任职去重派生的管理员钱包集合，不提供直接变更 extrinsic。
- 机构的主账户、费用账户和其它业务账户共享机构层有效管理员与岗位任职事实；具体账户能执行什么操作，由对应业务模块硬规则决定。
- 固定创世机构岗位定义与法定席位受治理骨架保护，任职可以依法轮换，但不得直接编辑管理员钱包集合。
- 机构管理员变化必须由创世、注册登记或具体选举/任免业务结果驱动，经 entity 同事务更新岗位任职与派生 admins。
- 个人多签继续由 `personal-manage` + `personal-admins` 独立管理，可以按个人多签规则变更管理员集合，不使用机构岗位模型。
- 内部投票引擎只负责投票、快照、计票和终态判定，不成为机构岗位、任职或管理员的第二真源。

## 禁止恢复

- 不得恢复机构 `AdminSetChangeAction`、public/private admins 管理员集合变更 call 或机构“换管理员”通用入口。
- 不得恢复机构管理员 `creator`、`created_at`、`updated_at`、姓名、公民 CID、岗位或来源的内嵌副本。
- 不得把个人多签管理员变更规则套用到机构。
