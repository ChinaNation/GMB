# ADR-008 SFID 省管理员三槽自治

- 状态:已采纳
- 日期:2026-05-01

## 背景

SFID 系统的省级治理需要从集中式管理改为省级自治。每个省维护一主两备
三个省管理员槽位,并由当前主管理员负责添加、更换备用管理员和主备交换。

## 决策

- 每省固定三个槽位:`main`、`backup_1`、`backup_2`。
- 主管理员可以添加、替换备用管理员。
- 主管理员把备用管理员设置为主管理员时,原主管理员自动降级为备用管理员。
- 省管理员签名私钥由该管理员本人登录时自动生成,也可在登录后手动生成或更换。
- 链上真相为 `ShengAdmins[Province][Slot]` 与
  `ShengSigningPubkey[Province][AdminPubkey]`。
- SFID 系统不再维护独立的省管理员链交互目录;需要链交互时放入
  `sheng_admins/chain_*.rs`。

## 影响

- 后端只保留 `ShengAdmin` / `ShiAdmin` 两个管理员角色。
- 登录模块只负责认证、会话和本人签名密钥加载。
- 省管理员名册、签名密钥生成/更换、备用管理员维护归 `sheng_admins/`。
- 公民、机构、CPMS 等业务继续在各自功能目录内处理权限与范围。

## 验收

- 省管理员页面展示一主两备。
- 本人签名密钥状态可查看、生成、重新生成。
- 主管理员可维护备用管理员并执行主备交换。
- 相关代码、文档和任务索引只描述当前二角色与三槽自治模型。
