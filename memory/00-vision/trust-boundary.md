# GMB 信任边界

## 1. 整体链路

```text
OnChina 注册局能力（身份录入、公民护照、账户绑定、凭证签发）
        ↓
CitizenChain（链上资格、治理、资产、交易）
        ↑
CitizenApp / CitizenWallet（在线交互、离线签名、投票、转账）
```

AI 开发系统不进入实名和权限信任链本身，只作为研发和治理基础设施存在。

## 2. OnChina 边界

- 负责注册局身份录入
- 负责公民护照发放
- 负责链账户绑定
- 负责凭证签发与认证授权
- 不保存原始实名数据

## 3. CitizenChain 边界

- 负责链上资格、治理、资产、交易规则执行
- 不保存普通公民的原始实名档案、护照号、出生日期、住址等非公开隐私信息
- 允许保存依法公开或公民主动进入公开身份场景的必要身份事实，包括机构法定代表人、机构岗位任职和竞选公开资料
- 公开身份事实不得扩张为公民原始实名档案或其他非公开信息
- 不承载 AI 推理逻辑
- 节点程序和节点 UI 对外属于一个桌面产品

## 4. CitizenApp 边界

- 负责钱包签名、认证接入、提案、投票、转账、permit 使用
- 是用户入口，不是信任根
- 不负责实名裁决
- 不负责链上规则定义
- 公开用户资料由钱包账户统一标识，资料元数据与媒体可按既定 Cloudflare R2/D1 契约公开读取；通讯录私人名称和联系人关系不属于公开资料。
- 通讯录明文与解密密钥只存在用户设备；Cloudflare 只保存端侧 AES-256-GCM 密文和不可逆 HMAC 联系人 ID，用于同一热钱包跨设备恢复，不能成为联系人关系明文真源。
- Cloudflare Session 证明设备子钥控制的钱包身份，不证明链上账户存在、余额充足或具备任何链上业务资格；链上资格必须由具体业务重新校验。

## 5. AI 系统边界

- 负责读取项目记忆、拆解任务、编写代码、执行测试、更新文档、辅助发布
- 不负责改变业务信任边界本身
- 不允许在不确认的情况下修改关键安全规则

## 6. 不可突破的红线

- OnChina 不保存原始实名
- 链上不保存普通公民的原始实名档案和非公开隐私信息
- 机构完整生效后，法定代表人姓名、公民 CID 号和唯一钱包账户属于机构法定公开信息，必须上链
- 创世只能写入仓库中已有的真实人员资料；没有真实任免结果时必须保持“法定代表人尚未任命”，不得编造姓名、CID、账户或以 `admins[0]` 回退
- 凭证和登录 challenge 必须短期有效
- Cloudflare 不得持有通讯录解密密钥或联系人账户/私人名称明文；用户注销时必须硬删除通讯录密文。
