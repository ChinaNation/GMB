# GMB 信任边界

## 1. 整体链路

```text
CPMS（离线实名签发）
        ↓ 二维码
SFID（在线验证、身份生成、账户绑定、permit）
        ↓
CitizenChain（链上资格、治理、资产、交易）
        ↑
WuMinApp（钱包、认证、投票、转账）
```

AI 开发系统不进入实名和权限信任链本身，只作为研发和治理基础设施存在。

## 2. CPMS 边界

- 负责实名录入、本地加密存储、多管理员审核、审计日志、二维码签发
- 永不联网
- 不负责在线认证
- 不负责链上交互

## 3. SFID 边界

- 负责二维码验签
- 负责在线身份生成
- 负责链账户绑定
- 负责 permit 签发与认证授权
- 不保存原始实名数据
- 不替代 CPMS 成为实名信任根

## 4. CitizenChain 边界

- 负责链上资格、治理、资产、交易规则执行
- 不保存真实身份
- 不保存原始实名信息
- 不承载 AI 推理逻辑
- 节点程序和节点 UI 对外属于一个桌面产品

## 5. WuMinApp 边界

- 负责钱包签名、认证接入、提案、投票、转账、permit 使用
- 是用户入口，不是信任根
- 不负责实名裁决
- 不负责链上规则定义

## 6. AI 系统边界

- 负责读取项目记忆、拆解任务、编写代码、执行测试、更新文档、辅助发布
- 不负责改变业务信任边界本身
- 不允许在不确认的情况下修改关键安全规则

## 7. 不可突破的红线

- CPMS 永不联网
- 二维码不包含实名原始数据
- SFID 不保存原始实名
- 链上不保存真实身份
- permit 必须短期有效
