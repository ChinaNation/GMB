# GMB 双聊天窗口接入规则

## 1. 目标

在保留现有 Codex 使用方式的前提下，新增 Claude 聊天窗口作为等价入口。

统一要求：

- Codex 与 Claude 都属于 GMB AI 编程系统入口
- 两个入口共用同一套 `memory/`
- 两个入口共用同一套任务卡
- 两个入口共用同一套模块边界与安全规则

## 2. 入口定义

- `Codex`：聊天入口之一
- `Claude`：聊天入口之一
- 当前承接任务的聊天窗口：本次任务的主入口

主入口职责：

- 先做需求分析
- 创建或更新任务卡
- 决定是否分派专业工作线程
- 在代码修改后同步更新文档
- 对包含 `检查为什么报错` 的只读报错诊断请求，直接检查并输出原因，不创建任务卡

## 3. 统一协议

Codex 与 Claude 都必须遵守：

1. 首轮先做需求分析
2. 得到继续信号后先创建任务卡
3. 代码更新后必须更新文档
4. 关键结论必须回写 `memory/`
5. 不允许依赖聊天历史作为长期记忆

例外：包含 `检查为什么报错` 的只读报错诊断请求不进入开发执行，因此不创建任务卡、不修改代码。

## 4. 长期记忆

长期记忆统一位于：

- `memory/00-vision/`
- `memory/01-architecture/`
- `memory/03-security/`
- `memory/04-decisions/`
- `memory/05-modules/`
- `memory/07-ai/`
- `memory/08-tasks/`

任何入口都不得在聊天窗口内维护独占规则。

## 5. 当前改造结论

从现在开始，GMB AI 编程系统不再把 Claude 仅定义为后台 Review Agent。

统一改为：

- Codex 可以作为主开发入口
- Claude 也可以作为主开发入口
- 两者只是不同聊天入口，不是两套系统
