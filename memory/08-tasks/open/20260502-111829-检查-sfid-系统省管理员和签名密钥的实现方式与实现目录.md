## 实施记录

- 任务卡已创建
- 2026-05-02 只读检查记录：
  - 省管理员模型已按 ADR-008 改为 `SHENG_ADMIN / SHI_ADMIN` 二角色，`KEY_ADMIN` 角色在代码和数据库迁移中已废止。
  - 省管理员 main 公钥仍来自 `sfid/backend/src/sfid/province.rs` 的 43 省常量；backup_1 / backup_2 目标上由链上 `ShengAdmins[Province][Slot]` 持久化。
  - SFID 后端已新增 `sheng_admins/signing_cache.rs`、`sheng_admins/bootstrap.rs`、`store_shards/sheng_signer.rs`，用于按 `(province, admin_pubkey)` 生成、加密落盘、登录态缓存省级签名 keypair。
  - 签名 seed 落盘路径为 `storage/sheng_signer/{province}_{admin_pubkey_hex}.enc`，算法为 AES-256-GCM，wrap key 由 `SFID_MASTER_KEK_HEX` 经 HKDF 按 admin_pubkey 派生；开发环境可 fallback 到 `SFID_SIGNING_SEED_HEX`。
  - `chain/sheng_admin/` 已有 add/remove backup 的真实推链编码与 HTTP handler；`chain/sheng_signer/` 已有 activate/rotate 的真实推链编码与 HTTP handler。
  - 当前关键未闭合点：`chain/sheng_admin/query.rs` 仍是常量 main + backup 空槽占位；4 个 push extrinsic 当前用 `sheng_signer_cache` 中的 signing pair 代签，而链端要求 admin 私钥签名，需 step2e 冷钱包签名通路接通后才是真正端到端可用。
  - 前端 `src/views/sheng_admin/` 已有名册、激活、rotate 页面，但 API 类型与后端响应存在不一致风险：后端 roster 返回 rows 数组，前端按 `{ province, entries }` 读取；后端 add/remove 返回 `{ ok, tx_hash }`，前端按 `RosterEntry` / `{ removed }` 读取。
