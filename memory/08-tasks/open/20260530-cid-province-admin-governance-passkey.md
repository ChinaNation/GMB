# CID 联邦注册局机构管理员治理 Passkey 与公民钱包签名第 1 步任务卡

## 任务目标

修复联邦注册局机构管理员治理安全模型：联邦注册局机构管理员只负责管理员治理，不再持有或使用云端业务签名私钥；联邦注册局机构管理员写操作必须通过登录态、Passkey、`QR_V1` 冷钱包 sr25519 挑战签名。

## 固定约束

- CID 生产 Passkey 域名固定为 `cid.crcfrcn.com`;开发环境可用 `localhost` 独立注册开发 Passkey。
- 全仓库只允许 `QR_V1` 一个二维码协议。
- 不新增 `CID_ADMIN_ACTION_V1` 或其它协议。
- 不做旧 signer 兼容，不迁移旧 seed，不读取旧路径。
- 不使用的代码、注释、文档必须清理。

## 范围

- 查看市注册局机构管理员列表：保留登录态访问。
- 新增、删除市注册局机构管理员：必须登录态 + Passkey + 当前联邦注册局机构管理员公民钱包签名。
- 编辑市注册局机构管理员姓名：只需要登录态，后端仍校验角色和省域。
- 新增、删除联邦注册局机构管理员：必须登录态 + Passkey + 当前联邦注册局机构管理员公民钱包签名。
- 编辑联邦注册局机构管理员姓名：只需要登录态，后端仍校验角色和省域。
- 删除新增联邦注册局机构管理员：必须由本省初始联邦注册局机构管理员通过 Passkey + 公民钱包签名提交。
- 删除联邦注册局机构管理员云端代签逻辑。
