# 任务卡：机构岗位字段与护宪大法官成员解析

## 当前状态

- 状态：旧管理员资料模型已废弃；护宪成员解析已按 entity 岗位任职完成
- 最终成果并入 `20260628-institution-admin-field-model-onchain.md`

## 当前有效模型

- 机构 admins 保存 `admin_account + family_name + given_name` 管理员人员集合；岗位与任职独立保存。
- NJD 岗位目录归 public entity：7 名护宪大法官、1 名首席大法官、2 名次席大法官、5 名大法官。
- 护宪终审成员从 NJD 的“护宪大法官”有效岗位任职读取，必须恰好 7 席；不能按管理员资料字符串过滤。
- 固定岗位代码、名称和席位由创世映射写入，并由 Node Guard 保护；依法换届只替换任职，不修改固定岗位定义。
- 姓名、公民 CID、岗位、任期和来源不再内嵌进 admins，也不建立第二份成员名册。
- 法定代表人是机构公开信息的独立三字段，不等于护宪成员，也不是创世必填项。
