# CitizenApp 通讯录操作与账户边界统一

## 任务需求

- 联系人三点菜单按“转账、私信、修改名称、删除联系人”排序，删除项使用红色文字。
- 修复修改联系人名称对话框在取消或保存后的异常渲染。
- 转账动作打开链上支付并预填联系人钱包账户；私信动作复用统一一对一聊天入口。
- “我的 → 通讯录”和“交易 → 通讯录”始终读取默认用户的同一份通讯录。
- 交易页右侧钱包只选择付款钱包，不得改变通讯录所属用户。

## 业务边界

- 通讯录所属用户唯一来源是 `WalletManager.getDefaultWallet()`。
- 当前付款钱包只决定签名付款账户；联系人账户只作为收款地址或聊天对方账户。
- 转账动作只预填收款地址，不自动填写金额、签名或提交。
- 私信复用 `openDirectChat()`，不新增聊天协议、存储或传输流程。
- 不修改 Cloudflare 接口、数据库结构、CitizenChain runtime 或 CitizenWallet。

## 预计修改目录

- `citizenapp/lib/my/user/`：统一通讯录默认用户来源，修改联系人菜单、改名对话框和动作接线；涉及代码、中文注释和残留清理。
- `citizenapp/lib/transaction/onchain-transaction/`：解除通讯录与当前付款钱包耦合并保留收款地址回填；涉及代码和注释。
- `citizenapp/lib/chat/`：只复用现有私信入口，不新增第二套聊天流程。
- `citizenapp/test/user/`、`citizenapp/test/ui/`：在现有测试中覆盖菜单、改名、转账、私信和双钱包隔离。
- `memory/05-modules/citizenapp/`：更新用户、交易和聊天技术文档，清理旧口径。

## 验收标准

- 两个入口显示默认用户的同一份通讯录，切换付款钱包不会改变联系人列表。
- 修改中文名称取消和保存均无乱码、无 Flutter 异常。
- 菜单顺序及红色删除项符合需求。
- 转账正确预填收款地址且不改变付款钱包；私信进入正确用户聊天窗口。
- 目标静态检查、自动测试和 Android 真机页面验收通过。
- 文档、中文注释和旧参数、旧文案、重复逻辑残留全部收口。
