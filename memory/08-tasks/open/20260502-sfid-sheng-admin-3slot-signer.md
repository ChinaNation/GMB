# SFID 省级管理员一主两备与签名密钥交互

## 任务背景

注册局-省级管理员页面需要按业务规则展示一主两备 3 个管理员槽位。当前只有主管理员是代码内置，备用管理员由主管理员添加、更换，也可以被主管理员设置为主管理员；设置为主管理员时，原主管理员自动降级为该备用槽位。

省级管理员签名私钥只归管理员本人所有。备用管理员的签名私钥不能由主管理员代生成，只能在该备用管理员本人登录时自动生成/加载，或由该备用管理员本人手动生成/更换。

## 任务目标

- 注册局-省级管理员页面显示主管理员、备用管理员 1、备用管理员 2 三个板块。
- 每个板块显示状态、生成时间、账户。
- 在状态后显示签名密钥操作按钮：未生成显示“生成”，已生成显示“更换”。
- “生成/更换”只允许当前登录管理员操作自己的签名密钥。
- 主管理员可添加、更换备用管理员，并可把备用管理员设置为主管理员。
- 备用管理员升主时执行槽位交换，原主管理员自动降级到该备用槽位。
- 检查并修复“省管理员已登录但签名密钥未生成/未加载”的状态来源问题。

## 改动范围

- `sfid/backend/sheng_admins/`
  - 省管理员名册、签名密钥状态、生成/更换、升主交换相关接口。
- `sfid/backend/login/`
  - 省管理员登录后签名密钥 bootstrap 检查。
- `sfid/frontend/sheng_admins/`
  - 注册局-省级管理员页面、一主两备展示、签名弹窗和操作按钮。
- `memory/`
  - 更新任务、架构或规则文档。

## 边界规则

- 主管理员可以管理备用管理员名册。
- 主管理员不能替备用管理员生成或更换签名私钥。
- 备用管理员本人可以登录自动生成/加载，也可以手动生成/更换自己的签名私钥。
- 签名密钥操作需要弹窗签名，不允许后端代替管理员账户签名。
- 不单独新建跨功能 API 或 chain 目录，省管理员链交互继续留在 `sheng_admins/` 模块内。

## 验收要点

- 注册局-省级管理员页面展示 3 个槽位。
- 未设置备用管理员显示账户 `-`、生成时间 `-`、状态 `未设置`。
- 当前登录管理员只能对自己的签名密钥看到 `生成/更换`。
- 主管理员能看到备用管理员管理操作。
- 备用管理员升主后，原主管理员自动进入对应备用槽位。
- SFID 后端能清楚区分本地 seed、进程 cache、链上激活状态。
- 改代码后补充中文注释、更新文档、清理旧入口和残留。
