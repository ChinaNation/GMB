# 历史和解与和平建国基金专项发行

## 任务需求

增加一种新的专项发行类型：发行一笔专用资金，用于赔偿历史中受到共产党迫害、受到政治运动迫害、蒙受国家灾难的人们及其相关家庭，并可用于修建、维护类似纪念场馆、档案馆、公共纪念设施或教育项目。

暂定中文名称：历史和解与和平建国基金。

推荐定位为 CitizenChain 上的“专项限制发行”，资金不得进入普通国库或普通流通拨款池，而应进入受链上规则约束的专项基金账户。该基金只能用于：

- 个人或家庭赔偿。
- 公共纪念、档案、教育、纪念场馆等项目。

## 必须遵守

- 不可突破模块边界。
- 不可擅自修改安全红线。
- 不可在链上保存真实身份、原始实名信息、受害经历、证据材料、家庭关系或政治标签。
- 不可由发行模块、赔偿模块或项目模块实现、复刻、绕过或内嵌任何投票流程。
- 基金设立、追加发行、用途调整、暂停、关闭和重大拨款规则必须调用既有投票引擎完成。
- 业务模块只允许调用投票引擎接口创建或绑定投票，不得自行处理人口快照、投票资格、联合签名、投票状态推进、计票、通过或否决判定。
- 若新增链上 call、交易载荷、storage key、pallet index、call index 或 QR 签名展示字段，必须先更新统一协议文档，并同步适配 `wuminapp` 与 `wumin`。
- 不默认设计任何历史兼容、过渡兼容、双轨兼容或旧交易载荷兼容；如确需兼容，必须由用户明确要求。
- 关键逻辑必须补中文注释。
- 改代码后必须更新文档并清理残留。

## 建议设计方向

### 1. 专项发行

在发行模块中新增专项发行类型，建议英文内部名先按简短语义使用 `ReconciliationFund`，中文说明为“历史和解与和平建国基金”。

该发行不是普通增发，不直接进入普通账户，而是进入专项基金账户。专项基金账户必须具备：

- 总额上限。
- 已发行金额。
- 已赔付金额。
- 已拨付项目金额。
- 冻结或暂停状态。
- 关闭或清算状态。

### 2. 专项账户

基金资金进入链上专项账户，支出时必须绑定用途：

- `个人或家庭赔偿`
- `公共纪念项目`

每笔支出必须能审计到 `claim_id` 或 `project_id`，但不得暴露真实身份和敏感经历。

### 3. 赔偿资格

赔偿资格建议采用“链下认定、链上验凭证”：

- 链下可信机构负责材料审核、身份确认、受害事实归档和资格裁定。
- 链上只接收资格凭证哈希、领取账户、赔偿等级、签名和防重放字段。
- 链上只记录领取状态、金额和匿名 claim 标识。

### 4. 纪念项目

纪念场馆、档案、教育、纪念设施等项目应单独建项目拨款路径：

- 项目提案。
- 预算总额。
- 分阶段拨款。
- 里程碑。
- 审计状态。
- 终止或追回规则。

项目拨款不得与个人赔偿混用。

### 5. 治理控制

以下事项必须进入治理流程：

- 基金设立。
- 首次专项发行总额。
- 是否允许追加发行。
- 追加发行额度。
- 赔偿资格规则。
- 赔偿等级规则。
- 公共纪念项目预算比例或上限。
- 基金暂停、恢复、关闭、清算。

## 待确认问题

- 该基金是一次性发行，还是允许治理追加发行？
- 首次发行总额如何确定？
- 赔偿对象是否包括本人、遗属、失踪者家属、被株连家庭和被没收财产家庭？
- 赔偿金额是统一金额、分级金额，还是按案件裁定金额？
- 资格认定由哪个机构或哪些机构负责？
- 资格认定是否允许申诉、复核和撤销？
- 纪念场馆、档案馆、教育项目的资金比例是否固定？
- 公共纪念项目是否允许地方机构、民间机构或受害者团体申请？
- 个人赔偿与公共项目是否共用同一个基金余额，还是分设子账户？
- 基金关闭后剩余资金如何处理？
