# SFID 注册机构多签地址派生加入 name 字段

- 创建时间：2026-04-05
- 状态：待执行
- 模块：Blockchain Agent + Mobile Agent
- 优先级：中

## 背景

> 2026-04-20 更新：域前缀已统一为 `DUOQIAN_DOMAIN = b"DUOQIAN_V1"` + `OP_MAIN = 0x00`，见 `primitives::core_const`。
>
> 2026-04-21 更新：`derive_duoqian_address_from_sfid_id(sfid_id, name)` 已重构为 `derive_institution_address(sfid_id, role)` + `role_from_name(name)`。name="主账户"/"费用账户" 走 `Role::Main`/`Role::Fee`（preimage 不含 name），其他非空 name 走 `Role::Named(name)` 并使用新 `OP_INSTITUTION = 0x05`。见任务卡 [20260421-op-institution-role-split](../done/20260421-op-institution-role-split.md)。

当前派生公式：`Blake2b_256(DUOQIAN_DOMAIN || OP_MAIN || SS58 || sfid_id)`，一个 sfid_id 只能派生一个地址。需要支持同一机构（同一 sfid_id）创建多个多签账户（如运营账户、备用金账户），通过 name 区分。

## 方案

`derive_duoqian_address_from_sfid_id` 函数加 name 参数：
- name 非空时参与派生：`Blake2b_256(DUOQIAN_DOMAIN || OP_MAIN || SS58 || sfid_id || name)`
- name 为空时保持原结果：`Blake2b_256(DUOQIAN_DOMAIN || OP_MAIN || SS58 || sfid_id)`

治理机构不传 name，地址不变，不需要重新创世。
