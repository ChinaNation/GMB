# 任务卡：注册局管理员列表表格化与联邦注册局全量目录

## 任务需求

注册局管理员列表从黑色管理员卡片改为普通表格。联邦注册局使用 `序号 / 省份 / 姓名 / 余额 / 操作`；市注册局使用 `序号 / 城市 / 姓名 / 余额 / 操作`。操作列中的 passkey 按钮统一显示为 `密钥`。点击除操作列之外的管理员行时，弹出该管理员完整信息。

联邦注册局管理员列表从“当前登录省份 5 人组”改为展示全部联邦注册局管理员；当前登录省份 5 人置顶，且只有本省 5 人的“更换”按钮可操作，其它省份的“更换”按钮禁用。后端仍必须做同省校验，前端禁用只作为体验提示。

## 预计修改目录

- `citizenchain/onchina/frontend/admins/`
  - 用途：改注册局管理员列表表格列、行点击详情弹窗、passkey 按钮文案和同省更换禁用逻辑。
  - 边界：只改注册局业务列表；非注册局本机构管理员卡片墙继续使用卡片展示。
  - 类型：代码与注释；同步清理黑色卡片在注册局表格中的残留调用。

- `citizenchain/onchina/src/auth/`
  - 用途：联邦注册局管理员接口返回全量省级组，并维护本地省份映射缓存供同省更换校验使用。
  - 边界：权限真源仍是链上 `PublicAdmins::FederalRegistryProvinceGroups`；本地只缓存展示与校验所需的省份归属。
  - 类型：代码与注释；不新增独立授权真源。

- `citizenchain/onchina/src/core/`
  - 用途：补 OnChina 侧链读辅助函数，批量读取全部联邦注册局省级管理员组。
  - 边界：只改 OnChina 链读适配层，不修改 `citizenchain/runtime/`。
  - 类型：代码与注释。

- `memory/`
  - 用途：记录任务执行结果、架构说明和残留清理结论。
  - 边界：只更新本任务相关文档。
  - 类型：文档。

## 实施步骤

1. 创建任务卡，确认前后端影响范围。
2. 后端新增全量联邦注册局省级组读取与省份映射缓存。
3. 前端注册局管理员列表表格化，并加入完整信息弹窗。
4. 更新文档、清理注册局列表黑色卡片残留。
5. 执行编译、构建和真实运行态验收。

## 验收标准

- 联邦注册局管理员列表显示全部省份管理员，本省 5 人置顶。
- 联邦注册局表头为 `序号 / 省份 / 姓名 / 余额 / 操作`。
- 市注册局表头为 `序号 / 城市 / 姓名 / 余额 / 操作`。
- 注册局列表不再渲染黑色管理员卡片。
- passkey 操作按钮显示 `密钥`。
- 点击非操作列区域弹出完整管理员信息。
- 非本省联邦注册局管理员的“更换”按钮不可操作，后端同省校验仍生效。
