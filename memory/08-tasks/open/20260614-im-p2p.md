# 任务卡：citizenapp P2P IM 基础落地

- 任务编号：20260614-im-p2p
- 状态：done
- 所属模块：citizenapp / citizenchain node
- 当前负责人：Codex
- 创建时间：2026-06-14

## 任务需求

在区块链节点软件和 citizenapp 中新增 `im` 目录，专门用于 P2P 即时通讯能力；citizenapp 底部在“多签”和“交易”之间新增“信息”Tab；通信全节点是私人通信节点，只服务自己的手机和自己的收件箱，不互为中继；钱包账户作为用户可见聊天账户，并支持聊天窗口内发起公民币转账。

## 本轮执行范围

- 新增 IM 架构 ADR，冻结私人通信全节点、钱包聊天账户、IPv4/IPv6 双栈、禁止互为中继等边界。
- 新增 citizenapp `lib/im/` 基础模型、信息 Tab 壳、支付提示模型、传输抽象、存储占位。
- 新增 citizenchain node `src/im/` 私人通信全节点边界模块和 Tauri 查询命令。
- 更新 IM 技术文档、citizenapp 总文档、node 技术文档、统一协议和统一命名登记。
- 本轮不落 Isar schema、不接入真实 OpenMLS、不实现真实跨节点投递和近场原生模块。

## 必须遵守

- 使用中文沟通，代码补中文注释。
- IM 通信不上链、不进 CID、不使用中心服务器。
- 通信全节点只服务自己，不做公共 Relay / DHT / rendezvous，不替别人存消息。
- 钱包账户是聊天账户和公民币收付款账户，但钱包私钥不得作为 IM 端到端加密密钥。
- 真实 OpenMLS、sc-network 能力、Isar schema 和近场原生实现进入后续任务前必须单独确认。

## 验收标准

- citizenapp 底部导航出现“信息”Tab，位置在“多签”和“交易”之间。
- “信息”Tab 展示 IM 私人节点、钱包聊天账户、支付提示和近场入口的基础界面，不误导为真实通信已完成。
- node 侧存在 `src/im/` 模块，能通过 Tauri 命令返回私人通信全节点边界。
- 文档同步记录最新技术方案，清理旧的“互为中继”或“Matrix 主选”口径。
- 运行格式或静态检查，记录验证结果。
