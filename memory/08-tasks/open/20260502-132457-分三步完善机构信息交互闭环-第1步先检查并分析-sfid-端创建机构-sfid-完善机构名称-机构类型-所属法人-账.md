## 技术方案：取消备案，改为 SFID 机构信息签名凭证

### 目标

- 不做备案按钮、备案状态、备案链上记录。
- SFID 创建并完善机构信息后，区块链端可以直接查询该机构。
- 区块链端正式注册机构时，只接收并校验 SFID 证明过的三类业务信息：
  - `sfid_id`
  - `institution_name`
  - `account_names[]`
- 链端不得要求、存储或校验 `a3/sub_type/parent_sfid_id` 等 SFID 内部机构资料。

### SFID 侧查询边界

- SFID 后台管理接口 `/api/v1/institution/*` 继续服务内部管理，可看到未完善草稿机构。
- 区块链端公开接口只返回"可注册机构"：
  - 必须有 `institution_name`。
  - 账户列表至少包含 `主账户`、`费用账户`。
  - 如果 SFID 内部要求 SFR 设置机构类型、FFR 设置所属法人，则这些条件只作为 SFID 侧是否允许公开/签凭证的内部门槛，不作为链端 payload 字段。

### 推荐 API

1. `GET /api/v1/app/institutions/search?q=xxx&limit=20`
   - 用途：区块链端按机构名称或 SFID 搜索。
   - 返回最小列表：
     - `sfid_id`
     - `institution_name`

2. `GET /api/v1/app/institutions/:sfid_id/registration-info`
   - 用途：区块链端准备正式注册时拉取 SFID 证明。
   - 返回：
     - `sfid_id`
     - `institution_name`
     - `account_names[]`
     - `register_nonce`
     - `province`
     - `signer_admin_pubkey`
     - `signature`

### 签名凭证语义

- `signature` 是 SFID 省级签名密钥对"机构注册信息"签名，不是业务字段。
- 签名只证明三项业务信息来自 SFID 且未被链端用户篡改。
- 推荐 SCALE payload 顺序：
  1. `DUOQIAN_DOMAIN`
  2. `OP_SIGN_INST`
  3. `genesis_hash`
  4. `sfid_id`
  5. `institution_name`
  6. `account_names[]`
  7. `register_nonce`
  8. `province`
  9. `signer_admin_pubkey`

### runtime 校验

- SFID 交互与凭证校验归属 `citizenchain/runtime/otherpallet/sfid-system/`。
- `sfid-system` 提供机构注册信息凭证校验能力,证明范围为：
  - `sfid_id`
  - `institution_name`
  - `accounts[].account_name`
  - `register_nonce`
  - `signature`
  - `province`
  - `signer_admin_pubkey`
- `duoqian-manage::propose_create_institution` 只负责注册执行:从 `accounts` 中提取账户名称列表,调用 `sfid-system` 的校验能力。
- `sfid-system` 使用 `(province, signer_admin_pubkey)` 查本模块中已激活的省级签名公钥并验签。
- `sfid-system` 验签通过后,`duoqian-manage` 才允许继续注册。
- 验签失败、nonce 重放、账户名称顺序或内容被改动，都拒绝。

### 不属于 SFID 证明范围

- 管理员列表。
- 阈值。
- 初始金额。
- 创建者资金 reserve。
- 投票流程。

这些仍归区块链 `duoqian-manage` 自己校验。
