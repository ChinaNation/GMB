# CitizenChain 44 个引导节点部署控制台

任务需求：
- 保留 CitizenChain 现有“运行 CI”和“运行 Release”功能。
- 扩展“部署区块链服务器”，先纳入 44 个引导节点，部署前必须选择目标节点。
- 每个节点可维护服务器 IP、引导节点密钥和验证节点密钥，并支持安全更换。
- 节点私密材料只允许保存到 macOS Keychain；生产操作必须经过 Touch ID。

安全边界：
- 网页、日志、配置文件和命令参数不得展示或持久化节点密钥明文。
- 控制台只显示密钥“已配置/缺失”，只允许覆盖更新，不允许读取旧值。
- 修改节点 IP、修改密钥和部署节点均属于生产操作，必须逐次完成 Touch ID。
- 未完成真实字段、格式、远端工作流和服务器路径核对前，不写入或部署节点密钥。

预计修改目录：
- `.github/workflows/`：让 CitizenChain CI 下载当前分支最新成功 WASM artifact，逐字节核对冻结 `:code` 后重建、校验并上传唯一正式创世状态包；涉及 CI 代码和旧空资源流程清理。
- `citizenchain/scripts/`：收紧 bake/prepack 创世包白名单，只允许 manifest 与 `chains/citizenchain/db`；涉及脚本代码、中文注释和残留清理。
- `deploy/actions/`：部署时由目标服务器直接下载 GitHub `main` 最新成功 CI 的 Linux amd 节点软件，并按普通升级或明确的新创世模式完成节点软件和真实服务验收。
- `deploy/`：扩展本地控制台、节点配置、Keychain 桥接和 CitizenChain 部署动作；源码由Git追踪，密钥和运行数据被精确忽略。
- `memory/08-tasks/open/`：记录任务范围、决策、进度和验收；会被 Git 跟踪。
- `memory/01-architecture/citizenchain/`：更新多节点部署架构与边界；会被 Git 跟踪。
- `memory/03-security/`：更新节点私钥与生产授权规则；会被 Git 跟踪。

验收标准：
- CitizenChain 卡片保留 CI、Release，并提供 44 个引导节点的选择与状态界面。
- 每个节点可安全修改 IP、引导节点密钥和验证节点密钥。
- 节点密钥逐节点隔离存储于 Keychain，任何 API 与日志均不返回明文。
- 所有节点生产变更和部署均通过 Touch ID 门禁。
- 使用真实本地控制台完成页面、Keychain 状态、认证拒绝和脱敏验收；不得误触发远端部署。
