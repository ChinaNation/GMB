# 府院机构地址接入治理框架

- 创建时间：2026-04-05
- 状态：待执行
- 模块：Blockchain Agent
- 优先级：中

## 背景

primitives/china 目录下 5 个府院机构数据文件（china_jy/lf/sf/zf/jc）当前仅作为制度预留：
- 地址被 china_zb.rs 汇总保护，防止通过 duoqian-manage-pow 抢注
- 不在 institution_org 识别范围内，无法参与内部投票或多签转账
- 无法通过 duoqian-manage-pow 注册（ReservedAddressChecker 拦截）

需要将这些机构正式接入治理框架，使其具备发起转账提案、内部投票等能力。

## 涉及机构

| 文件 | 机构 | 说明 |
|------|------|------|
| china_jy.rs | 公民教育委员会 | 1 个机构 |
| china_lf.rs | 联邦立法院 | 多个机构 |
| china_sf.rs | 联邦司法院 | 多个机构 |
| china_zf.rs | 联邦政府 | 多个机构 |
| china_jc.rs | 联邦监察院 | 多个机构 |

## 推荐方案：扩展 institution_org 识别范围

把府院机构当作和 NRC/PRC/PRB 同级的治理机构，新增 org 类型：

```
现有：ORG_NRC=0, ORG_PRC=1, ORG_PRB=2, ORG_DUOQIAN=3
新增：ORG_JY=4, ORG_LF=5, ORG_SF=6, ORG_ZF=7, ORG_JC=8
```

## 改动清单

| # | 文件/模块 | 改动 |
|---|----------|------|
| 1 | `internal_vote.rs` | `is_valid_org` 扩展，接受 ORG_JY..ORG_JC |
| 2 | `duoqian-transfer-pow` | `institution_org` / `institution_pallet_address` 加入 5 个新数组查找 |
| 3 | `admins-origin-gov` | 创世初始化加入府院机构的管理员列表 |
| 4 | `count_const.rs` | 新增府院机构的内部投票阈值、管理员数量常量 |
| 5 | `voting-engine-system` | `InternalThresholdProvider` / `InternalAdminCountProvider` 配置新 org 阈值 |
| 6 | `china_zb.rs` | 移除府院 main_address 的保护（已是正式机构，不再需要防抢注） |
| 7 | `configs/mod.rs` | RuntimeInternalAdminProvider / RuntimeInternalThresholdProvider 适配新 org |
| 8 | 节点前端 `proposal.rs` | institution_org 映射新增府院机构名称显示 |

## 不推荐的替代方案

走 duoqian-manage-pow 注册流程：需要放开 ReservedAddressChecker 保护，且机构变成 ORG_DUOQIAN 类型，阈值和管理员从链上动态读取，和府院的制度定位不匹配。

## 前置条件

- 确认各府院机构的管理员人数和内部投票阈值（需要和白皮书对齐）
- 确认是否参与联合投票（当前联合投票权重 105 = NRC:19 + PRC:43 + PRB:43，如果府院加入需要重新分配权重）

## 备注

本次改动为 runtime 逻辑变更，上链需要 runtime upgrade。
