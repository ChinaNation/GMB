# SFID 自动公权机构初始化与启动解耦

任务需求：
- 自动生成的公权机构只在首次初始化或行政区划变化时写入数据库。
- 后端正常启动不得全量生成、全量 upsert 公权机构，避免健康检查超时。
- 已持久化数据按省/市权限查询，需要时从数据库读取。
- 行政区变化时只更新对应省/市范围，保持既有 `sfid_number` 不变。

所属模块：
- SFID 后端 `gov`
- SFID 后端 `core`
- SFID 后端 `subjects`
- SFID 技术文档

输入文档：
- `memory/01-architecture/sfid/SFID_TECHNICAL.md`
- `memory/01-architecture/sfid/README.md`
- `memory/07-ai/task-card-template.md`
- 用户本轮技术方案确认内容

必须遵守：
- 不恢复 `backend/institutions`、`backend/src`、独立 `chain` 业务目录。
- 不引入第二身份键；机构唯一身份仍为 `sfid_number`。
- 不兼容旧全量启动流程，直接清理启动期全量写库残留。
- 市管理员只能查本市数据，联邦管理员只能查本省数据，后端必须带省/市范围查询。
- 代码必须补中文注释，改代码后必须更新文档并清理残留。

输出物：
- 后端启动流程修复。
- 自动公权机构显式初始化/局部对账入口。
- 持久化写入差异化，已有目标数据不重复写库。
- 文档更新。
- 残留清理与验证结果。

验收标准：
- `./sfid-run.sh` 不再因公权机构全量 upsert 阻塞健康检查。
- 空库可通过显式初始化生成自动公权机构。
- 已有库重启不重复 upsert 全部自动公权机构。
- 省/市局部对账只影响目标范围，行政区改名不改变既有 `sfid_number`。
- 公权机构列表仍按省/市权限从数据库直接查询。
- `cargo check` 通过。
