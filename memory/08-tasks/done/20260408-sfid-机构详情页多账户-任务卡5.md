## 交互流程

```
市详情页(InstitutionListTable)
  点"详情"按钮
  ↓
机构详情页(InstitutionDetailPage)
  ├─ 机构头部:SFID / 机构名 / 省市 / 分类
  ├─ AccountList:当前账户(sfid_id, account_name, duoqian_address, 状态)
  └─ "+ 新建账户" → CreateAccountModal
        ├─ 输入 account_name(自由文本,≤30 字)
        ├─ 前端校验同 sfid 下不重名
        ├─ 提交调 createAccount → 上链
        └─ 成功刷新 AccountList
```

## 验收

1. `npx tsc --noEmit` exit 0
2. 点机构表格行"详情"能进入新的详情页
3. 详情页能看到机构信息 + 账户列表(当前至少 1 条迁移过来的旧记录)
4. "+ 新建账户"弹窗输入账户名 → 成功后账户列表刷新
5. 同 sfid 下同名账户第二次创建返回 409 + 前端显示 message.error
6. 账户"删除"按钮软删账户(调 deleteAccount)

## 风险

- 删除账户按钮涉及 DELETE,要双重确认,避免误删链上已注册账户(本卡只做软删 sfid 系统记录,不触链)
- CreateAccountModal 和 InstitutionDetailPage 是两层嵌套,state 管理要小心
- App.tsx 改动尽量小,InstitutionsView 内部管理 selectedSfidId 即可
