## 任务需求

把 `duoqian-transfer-pow` **全部 3 组业务**(transfer / safety_fund / sweep)的"每管理员在线 `vote_X`"模式,统一改造为"发起人离线收齐 N 个管理员 sr25519 签名后一笔 `finalize_X` 代投"的离线 QR 聚合签名路径。

三组业务映射表:

| 组 | op_tag | from | to | remark |
|---|---|---|---|---|
| transfer | OP_SIGN_TRANSFER (0x15) | institution.main_address | beneficiary | 用户填 |
| safety_fund | OP_SIGN_SAFETY_FUND (0x16,新增) | NRC_ANQUAN_ADDRESS | beneficiary | 用户填 |
| sweep | OP_SIGN_SWEEP (0x17,新增) | institution.fee_address | institution.main_address | 空 |

三组**统一使用** `TransferVoteIntent` 结构 + 共用 `verify_and_cast_votes` helper,op_tag 做签名域硬隔离。

## 所属模块

- Blockchain Agent / `citizenchain/runtime/transaction/duoqian-transfer-pow`
- Blockchain Agent / `citizenchain/runtime/primitives`(新增 2 个 op_tag)
- Blockchain Agent / `citizenchain/runtime/src/configs/`(PowTxAmountExtractor 切换)

## 必须遵守

- 不改 `voting-engine-system` 一行
- 不改 runtime `RuntimeInternalAdminProvider` / `RuntimeInternalThresholdProvider`(Step 1 已就位)
- 不加兼容/过渡,旧路径物理删除
- 签名消息铁律:`DUOQIAN_V1 || op_tag || ss58_le || blake2(intent)` → blake2_256
- Admin sr25519 公钥 = AccountId32 字节,复用 `duoqian_manage_pow::Pallet::pubkey_from_accountid`
- 三组签名域严格隔离:跨组重放必须失败
- SweepAction 加 `proposer` 字段(满足 intent 构造需要)
