## 关键设计

### 三 tab 共享同一套视图组件

```tsx

interface Props {
  category: InstitutionCategory;      // 决定拉取哪类机构 + 锁弹窗字段
  tabLabel: string;                   // "公安局" / "公权机构" / "私权机构"

```

### 三角色 scope 分流(`ScopedEntry` 包装)

```tsx
<ScopedEntry
  renderProvinceList={() => <ProvinceGrid onPick={setSelectedProvince} />}
  renderProvinceDetail={(province) => <CityGrid province={province} onPick={setSelectedCity} />}
  renderCityDetail={(province, city) => <InstitutionListTable province={province} city={city} category={category} />}
/>
```

## 风险

1. **scope 语义和后端要一致**:前端跳过省/市列表的时机必须跟后端 `scope::filter_by_scope` 匹配,否则前端以为能看到的数据后端返回空,造成空页面。前端 `useScope` 和后端 `rules.rs` 字段命名已对齐,风险低
2. **City 清单需要从 sfid_meta 拉**:CityGrid 需要某省的所有市,走现有 `getSfidMeta` + `listSfidCities` 接口,不动后端
3. **App.tsx 改动范围**:本卡改动 App.tsx,风险是触发已有的页面引用现有 state(`institutionRows` 等),必须只替换 3 个 tab 的 body,不动其他 state
