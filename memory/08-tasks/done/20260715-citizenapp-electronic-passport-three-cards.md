# CitizenApp 电子护照三身份卡 UI

- 状态：done

任务需求：
- 电子护照页面始终显示“匿名访客”“公民 · 投票身份”“公民 · 竞选身份”三张卡。
- 当前身份对应卡片排在首位，右上角唯一显示“当前身份”，并显示该用户真实链上身份值。
- 非当前投票/竞选卡只显示对应身份字段名称，不显示用户数据、占位值或示例值。
- 匿名访客卡固定显示“没有公民身份信息”，彻底删除两种旧访客描述。
- UI 严格以用户确认的三状态高保真图为视觉基线。

安全与数据边界：
- 电子护照继续只读默认热钱包对应的 finalized `VotingIdentityByAccount` 与 `CandidateIdentityByAccount`。
- 链读取或解析失败不得静默降级为匿名访客；此时三卡均不标记当前身份，也不显示真实值。
- 非当前卡即使底层存在对应身份数据也不得显示真实值，避免重复暴露。
- 不修改 runtime、Cloudflare、D1、钱包协议、二维码协议或链上身份 SCALE 布局。
