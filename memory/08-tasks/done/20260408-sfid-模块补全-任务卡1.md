# 任务卡 1:sfid 后端模块补全 — 所有 SFID 工具归位到 `backend/src/sfid/`

- 创建日期:2026-04-08
- 模块归属:SFID Agent(`sfid/backend/sfid/` + 调用方同步改 import)
- 依赖:无
- 后续任务:任务卡 2(机构/账户两层模型),任务卡 3/4/5 见需求方案
- 优先级:高(所有后续任务的前置基础)

## 背景

当前 `sfid/backend/sfid/` 已经是独立模块,含 `generate_sfid_number` / `province` / `city_codes`,
但还有以下 SFID 工具散落在其他地方,导致"SFID 工具"这个概念不完整:

- `validate_sfid_number_format` 在 `federal-registry/institutions.rs:876`,`pub(super)` 可见性
- SubjectProperty 枚举(M/Z/N/G/S/F)没有独立类型,分散在 `sfid/mod.rs::resolve_subject_property`、前端 `sfid_meta` 接口里
- 机构码枚举(ZG/ZF/LF/SF/JC/JY/CB/CH/TG)散在 `resolve_org_type` 函数和 `app_core/runtime_ops.rs`
- 省份和城市清单已在 `sfid/province.rs` 和 `sfid/city_codes/`,但对外没有统一的高层 API `all_provinces()` / `cities_of(province)`
- **没有"机构分类"的概念**:后续任务卡 2 需要 `InstitutionCategory` 枚举
  (`PublicSecurity` / `GovInstitution` / `PrivateInstitution`),本任务卡先建好枚举占位

## 目标

把所有 SFID 相关的"工具函数 / 常量 / 枚举 / 分类逻辑"**全部**归位到 `backend/src/sfid/` 模块,
成为所有业务模块(`federal-registry/` / `operate/` / `business/` / `chain/` / `app_core/`)的**唯一共享依赖**。

本任务卡**不修改任何业务行为**,只做工具重组和 import 路径同步。

## 改动范围

### 1. `sfid/` 模块内新增 / 调整的子文件

```
backend/src/sfid/
├── mod.rs              — 重组:仅保留 pub use 导出,不放实现
├── generator.rs        — 从 mod.rs 搬出 generate_sfid_number + GenerateSfidInput
├── validator.rs        — ← 新增:validate_sfid_number_format + SFID_ID_* 常量(从 federal-registry/institutions.rs 搬过来)
├── province.rs         — 保留,追加 pub fn all_provinces() -> &'static [ProvinceEntry]
├── cities.rs           — ← 新增:pub fn cities_of(province: &str) -> Vec<CityEntry>
├── subject_property.rs               — ← 新增:#[derive] SubjectProperty 枚举 + resolve_subject_property / a3_label / a3_all
├── institution_code.rs — ← 新增:InstitutionCode 枚举(ZG/ZF/LF/SF/JC/JY/CB/CH/TG) + resolve + label
├── category.rs         — ← 新增:InstitutionCategory 枚举 + classify(subject_property, institution_code, name) 分类函数
├── admin.rs            — 保留不动
└── city_codes/         — 保留不动
```

每个子文件 **≤ 300 行**,如超出则再拆。

### 3. 新增的高层 API

```rust
// sfid/province.rs
pub fn all_provinces() -> &'static [ProvinceEntry];

// sfid/cities.rs
pub fn cities_of(province_name: &str) -> Option<Vec<CityEntry>>;

// sfid/subject_property.rs
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum SubjectProperty { K1=M, K1=Z, K1=N, K1=G, K1=S, K1=F }
impl SubjectProperty {
    pub fn from_str(s: &str) -> Option<Self>;
    pub fn as_code(self) -> &'static str;
    pub fn label_zh(self) -> &'static str;
}
pub fn all_subject_properties() -> &'static [SubjectProperty];

// sfid/institution_code.rs
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum InstitutionCode { ZG, ZF, LF, SF, JC, JY, CB, CH, TG }
impl InstitutionCode {
    pub fn from_str(s: &str) -> Option<Self>;
    pub fn as_code(self) -> &'static str;
    pub fn label_zh(self) -> &'static str;
}

// sfid/category.rs
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum InstitutionCategory {
    /// 公安局(K1=G + ZF + institution_name == "公民安全局")
    PublicSecurity,
    /// 公权机构(K1=G,非公安局)
    GovInstitution,
    /// 私权机构(K1=S / K1=F)
    PrivateInstitution,
}
pub fn classify(subject_property: SubjectProperty, code: InstitutionCode, institution_name: &str) -> Option<InstitutionCategory>;
```

## 风险

- **搬迁 `validate_sfid_number_format` 时可能触发其他地方的 `SFID_ID_*` 常量定义冲突**:需要全局 grep 确认这些常量只在 institutions.rs 里定义过
- **`SubjectProperty` / `InstitutionCode` 枚举新增后,本任务不强制切换现有 `resolve_subject_property(&str)` 的签名**,保持对外 `pub fn resolve_subject_property(s: &str) -> Result<&'static str, &'static str>` 的兼容,同时新增 `SubjectProperty::from_str` 作为未来使用。避免任务卡 1 牵动 `generate_sfid_number` 内部重构
- **`classify` 分类函数的分类规则**(PublicSecurity 判定依赖 institution_name == "公民安全局")在任务卡 1 里只是定义函数,**本任务不会有调用点**,真正用起来是任务卡 2
