### 边界规则

- `citizen_tab_page.dart` 承接原 `main.dart` 中的 `VotingPage` 与公民 Tab 内部切换逻辑，`main.dart` 只保留 App 壳、应用锁、底部导航。
- `vote/` 只负责“投票”二级 Tab 当前占位与后续公民投票入口扩展，不承接治理列表或机构详情。
- `governance/` 只负责“治理”二级 Tab 的全局提案列表、列表缓存、待投票红点统计。
- `institution/` 负责机构注册表、机构分类展示、机构详情、管理员列表、管理员激活、机构管理员链上查询。
- `proposal/` 负责所有具体提案的发起、详情、投票、链上编码提交；不同提案必须按三级目录拆分。
- `citizen/shared/` 放公民模块跨二级目录共用能力；`citizen/proposal/shared/` 放多个具体提案共用能力，禁止形成无边界大杂烩。

### 迁移顺序

1. 新建 `lib/citizen/` 目标目录结构，先不删除旧文件。
2. 从 `main.dart` 抽出 `VotingPage`、机构分类列表、机构卡片到 `citizen_tab_page.dart` 与 `institution/institution_list_page.dart`。
3. 搬迁 `institution_data.dart`、`institution_detail_page.dart`、`admin_list_page.dart`、`institution_admin_service.dart`、`activation_service.dart` 到 `institution/`。
4. 搬迁 `all_proposals_view.dart`、`proposal_cache.dart` 到 `governance/`。
5. 搬迁 `proposal_context.dart` 到 `citizen/shared/`，修正治理列表、机构详情、提案详情引用。
6. 搬迁通用投票能力到 `proposal/shared/`，包括 `internal_vote_service.dart`、`pending_vote_store.dart`、`proposal_vote_widgets.dart`，并抽出 `proposal_models.dart` 承接 `ProposalMeta`、`ProposalWithDetail` 等共用模型。
7. 搬迁转账提案到 `proposal/transfer/`，搬迁 Runtime 升级提案到 `proposal/runtime_upgrade/`。
