# 费率规则常量统一到 primitives::fee_policy

## 背景

[primitives/src/core_const.rs](citizenchain/runtime/primitives/src/core_const.rs) 当前混合了 5 类常量,其中"2. 交易手续费模型(Fee Model)"职责独立,应单独成文件,便于:
- 单一权威源:全链费率规则集中一处
- 命名空间清晰:`primitives::fee_policy::*` 替代 `primitives::core_const::ONCHAIN_*`
- 防漏改:所有费率引用从一个固定路径出
- 治理安全:费率改动只能通过修改本文件 + runtime 升级,**不允许走 storage 治理**

同时修复 2026-05-03 整改时简化引入的归类错误:
- `RuntimeCall::DuoqianTransfer(_)` 全部 1 元固定 → propose_X 三个应按 `amount × 0.1%`
- `RuntimeCall::ResolutionIssuance` propose 1 元 → 应免费(决议发行属货币政策)
- `RuntimeCall::ResolutionDestro(_)` 全部 1 元 → 应免费(决议销毁属货币政策)

## 边界

- **不新建 pallet**:费率规则纯库 const,无 storage、无 extrinsic、无治理入口
- **不引入 trait/路由架构**:沿用现有 `OnchainTxAmountExtractor` match RuntimeCall 形式
- spec_version 不变(用户拍板)
- chainspec.json 不动
- ABI 完全不变
- 链下 wuminapp / wumin / sfid 端不感知

## 4 类费用规则(用户拍板)

| 类别 | 规则 | 实际收费 |
|---|---|---|
| 免费 Free | NoAmount | 0 |
| 投票 Vote | FlatFee `VOTE_FLAT_FEE = 100 FEN` | 1 元 |
| 链上交易 OnchainTx | `max(amount × 0.1%, 10 FEN)` | 0.1 元起 |
| 未识别 Unknown | 拒绝 | 不入块 |

链下交易费率(`OFFCHAIN_FEE_RATE_MIN/MAX/MIN_FEE`)仍属本常量文件,但**不参与链上 extrinsic 计费**——它们由 offchain-transaction 模块自治使用(清算行通过 `propose_l2_fee_rate` 投票设置个体费率,在 [MIN, MAX] 区间内)。

### 4 类计费规则最终落地(`OnchainTxAmountExtractor`)
- **免费(NoAmount)**:System / Timestamp / ShengBankInterest / CitizenIssuance / ResolutionIssuance(全部) / ResolutionDestro(全部) / VotingEngine::finalize_proposal / OffchainTransaction::set_max_l2_fee_rate / DuoqianManage::register_sfid_institution
- **投票统一价(VOTE_FLAT_FEE = 1 元)**:VotingEngine 5 个投票入口 / SfidSystem 全部 / FullnodeIssuance bind+rebind / AdminsChange / RuntimeUpgrade / GrandpaKeyChange / DuoqianManage 其他 / OffchainTransaction 其他
- **链上交易(amount × 0.1%, ≥0.1 元)**:Balances 全部转账 / DuoqianManage propose_create_personal/institution/close / DuoqianTransfer 3 个 propose_X(propose_transfer/propose_safety_fund_transfer/propose_sweep_to_main) / OffchainTransaction deposit/withdraw/submit_offchain_batch_v2(sum fee_amount)
- **未识别(Unknown)**:Balances 兜底 + 其他 RuntimeCall 兜底,直接拒绝

### 设计铁律(写入 fee_policy.rs 模块文档)
1. 不可治理修改:`pub const`,改费率 = 改文件 + runtime 升级
2. 单一权威源:全仓只许从 `primitives::fee_policy::*` 导入
3. 链上/链下分离语义:`ONCHAIN_*` 由 onchain-transaction pallet 用,`OFFCHAIN_*` 由 offchain-transaction pallet 用
4. 单位统一 FEN(1 GMB = 100 FEN)

### spec_version / chainspec / ABI
- spec_version 不动(用户拍板)
- chainspec.json 不动
- 仅常量重组,extrinsic 签名/编码不变,链下端无感

## 全仓审计后增量整改(2026-05-03)

**P1 — `pallet_grandpa` 显式归类**:
- 原:`RuntimeCall::Grandpa(_)` 未匹配,落到 `_ => Unknown`,签名版 `report_equivocation` 被拒
- 改:`RuntimeCall::Grandpa(_) => NoAmount`(等价证据上报属公益,统一免费;unsigned 路径本就 bypass ChargeTransactionPayment)
- 副作用:补 Grandpa 后所有 RuntimeCall 变体已穷尽,**删除 `_ => Unknown` 兜底**(将来新增 pallet 若忘归类会编译期 non-exhaustive match 报错,强制开发者显式归类)

**P21 — offchain-transaction 影子常量收编**:
- `runtime/transaction/offchain-transaction/src/settlement.rs:46` 删除 `pub const MIN_FEE_FEN: u128 = 1;` → 直接 `primitives::fee_policy::OFFCHAIN_MIN_FEE`
- `runtime/transaction/offchain-transaction/src/fee_config.rs:19,22` 改 `L2_FEE_RATE_BP_MIN/MAX` 从 `OFFCHAIN_FEE_RATE_MIN/MAX.deconstruct() / 100_000` 推导(`Perbill::deconstruct` 是 `const fn`,编译期推导)
- `node/src/offchain/offchain_transaction/rpc.rs:344` 删除 `const MIN_FEE_FEN: u128 = 1;` → 直接 `primitives::fee_policy::OFFCHAIN_MIN_FEE`
- 新增 `fee_policy::tests::offchain_perbill_to_bp_conversion_stable` 单测:验证 Perbill→bp 换算关系不被破坏

**P22(UI 端硬编码)— 暂不改**:
- 节点 Tauri UI(`node/frontend/home/transaction/fee.ts`)和 wuminapp 移动端(`wuminapp/lib/rpc/onchain.dart`)的硬编码常量保留
- 技术原因:wuminapp 用 smoldot 内嵌轻节点,smoldot 不响应自定义 jsonrpsee handler;真正接入需暴露 Runtime API 必改 spec_version,违反铁律
- 用户决策:不改,保留 UI 预估硬编码
