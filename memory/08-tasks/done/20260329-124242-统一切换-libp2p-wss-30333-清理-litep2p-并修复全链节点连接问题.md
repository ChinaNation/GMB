# 任务卡：统一切换 libp2p + WSS 30333，清理 litep2p，修复全链节点连接问题

- 任务编号：20260329-124242
- 状态：open
- 所属模块：citizenchain / node / wuminapp / sfid（跨模块）
- 当前负责人：Claude

## 任务需求

将全链 P2P 网络从 litep2p + WS 30333 统一切换为 libp2p + WSS 30333。
节点启动时自动生成 TLS 自签证书并持久化，用户无需任何手动配置。
清理所有 litep2p 残留代码，只保留 libp2p 实现。
修复当前本地节点 0 peers 无法连接网络的问题。

## 背景

- 当前网络后端为 litep2p（Substrate SDK 新默认），但 litep2p 不支持 WSS 监听、自签证书、打洞（DCUtR）、中继（Relay）
- libp2p 原生支持上述所有能力，且 Substrate 已内置 libp2p 后端，只需启用
- 本地 node 节点缺少 `--listen-addr` 导致使用纯 TCP，无法连接服务器的 WS 端口
- 服务器节点 `on_finalize` 空块 panic 导致链卡死（#51），此为独立问题，本任务不处理出块问题

## 技术要点

### TLS 自签证书自动生成与持久化

```text
节点启动时：
1. 检查 <base-path>/tls/cert.der + key.der 是否存在
2. 不存在 → rcgen 生成 ed25519 自签证书，写入磁盘
3. 存在 → 直接加载
4. 将证书传入 libp2p WsConfig::set_tls_config()
5. 拨号端：自定义 rustls::ClientConfig，通过 peer ID 验证身份，跳过 CA 证书校验
```

### 自定义 Protocol ID

```text
当前日志：Using default protocol ID "sup" because none is configured
改动：在 chainspec 中配置专属 protocol ID "citizenchain"，避免与其他 Substrate 链混淆
```

## 风险点

| 风险 | 说明 | 应对 |
|------|------|------|
| smoldot-pow WSS 支持 | 原生平台完全不支持 WSS，需从零添加 rustls + TLS 握手 + 自签证书验证器 | 已确认改动范围：default.rs 中增加 WSS 连接分支，约 5 个文件 |
| sfid subxt WSS 连接 | sfid 用 subxt `from_insecure_url` 连 RPC，需信任节点的自签证书 | subxt 底层用 jsonrpsee，需自定义 rustls ClientConfig 跳过 CA 校验 |
| 44 台服务器同步部署 | 需要同时更新所有服务器，否则新旧协议不兼容 | 先在测试环境验证，再批量部署 |
| 证书持久化 | 证书必须持久保存，否则重启后已连接节点需要重新 TLS 握手 | 证书存储在 `<base-path>/tls/`，跟随链数据一起持久化 |
| DCUtR/Relay 集成 | Substrate 的 sc-network 未启用这些 libp2p 功能，需要在 SDK fork 中注册对应的 NetworkBehaviour | 参考 libp2p 官方 hole-punching tutorial，在 sc-network 的 behaviour 组合中加入 |
