# 任务卡 6:公安局机构跟 sfid 工具市清单对账

- 创建日期:2026-04-08
- 模块归属:SFID Agent(`sfid/backend` + `sfid/frontend`)
- 依赖:任务卡 1 / 2 / 3 / 4 / 5

## 核心铁律

1. 每个市恰好 1 个公安局,机构名称固定为 `"{市名}公安局"`
2. 机构 sfid_id 与机构名称**一对一**终身绑定,自动生成
3. sfid 工具的市清单是权威,公安局机构跟着市清单增/删/改
4. 机构和账户是**两套独立生命周期**:
   - 机构的增删改不触链
   - 链上只认账户(`register_sfid_institution` 的 `name` = account_name)

## 方案

### 对账函数(eager,按 city_code 主键)

`reconcile_public_security_for_province(store, province)`:

- **增**:sfid 工具有但 `multisig_institutions` 没有的市 → 生成新机构(`sfid_id` + `institution_name="{市名}公安局"` + `city_code`)
- **删**:`multisig_institutions` 有但 sfid 工具没有的 city_code → 删除机构记录(**不触链**,`multisig_accounts` 孤儿记录保留)
- **改名**:`city_code` 相同但 `city` 字符串不同 → 更新 `city` 和 `institution_name` 字段,`sfid_id` 不变

### 触发时机

- 后端启动时(43 省全量对账,幂等)
- 前端进入"公安局 - 某省"详情页时(调 POST `/api/v1/public-security/reconcile?province=...`)

### 数据模型变动

`MultisigInstitution` 新增 `city_code: String` 字段(2 位数字代码),作为对账主键。

### 后端启动迁移

老记录没 `city_code`,启动时按 `sfid::province::city_code_by_name(province, city)` 反查补齐。
补齐失败的(比如市名已被改动)保留空字段,对账时会被当成"新记录"误删 — 所以启动钩子先跑迁移再跑对账。

## 改动清单

### 后端
- `institutions/model.rs`:`MultisigInstitution` 加 `city_code: String`
- `institutions/service.rs`:`reconcile_public_security_for_province(store, province)`
- `institutions/handler.rs`:新 handler `reconcile_public_security`(POST)
- `app_core/runtime_ops.rs`:启动钩子 `backfill_and_reconcile_public_security`(对 43 省)
- `main.rs`:注册 `POST /api/v1/public-security/reconcile` + 启动调用钩子

### 前端
- `api/institution.ts`:`reconcilePublicSecurity(auth, province)`
- `views/institutions/InstitutionsView.tsx`:`PUBLIC_SECURITY` 分支在进入省详情页时先调 reconcile,然后直接显示机构表格(跳过市卡片层,没有 "+ 新增" 按钮)

## 不改

- 公权 / 私权 tab(保持三层模型)
- 链上代码
- `multisig_accounts` 层

## 验收

- `cargo build` 通过
- `npx tsc --noEmit` 通过
- 启动后端 → log 看到"对账完成 43 省"
- 后端 DB runtime_cache_entries 里的 multisig_institutions 出现每市一条的公安局记录(43 省 × N 市)
- 前端进公安局某省详情页 → 直接看到该省所有市公安局列表,没有"新增"按钮
- 点某行进详情页 → 可以新建账户(流程跟公权/私权一样)
