## 实施记录

- 任务卡已创建
- 已装载 `citizenchain/runtime` 上下文，并只读检查 `issuance` 下 4 个模块及相关 runtime feature 配置。
- `resolution-issuance::weights` 占位问题：存在。`src/weights.rs:3-4` 明确写着“先使用保守权重占位，后续必须通过 citizenchain/scripts/benchmark.sh 重新生成”，且当前文件没有 autogenerated benchmark 头。该文件内 4 个 `WeightInfo` 方法均为手写常量权重，应视为待正式 benchmark。
- “4 个模组都待办”口径：如果指 `issuance` 下 4 个 pallet，并非全部都自陈占位。`fullnode-issuance`、`citizen-issuance`、`shengbank-interest` 的 `weights.rs` 均有 benchmark CLI 生成头；但 `shengbank-interest` 在上一轮为 hook 手工补了两个权重函数，文档已记录发布前建议重新跑正式 benchmark。
- `fullnode-issuance` 的 `sp-std` 冗余问题：存在。`Cargo.toml:16` 声明 `sp-std`，`std` feature 也传播 `sp-std/std`，但 `rg` 仅在 Cargo 中命中，`src/lib.rs` 未实际使用。
- `citizen-issuance` try-runtime 传播问题：部分存在但无实际影响。该 crate 的 `try-runtime = []` 为空；runtime 根 `try-runtime` 已包含 `citizen-issuance/try-runtime`，但模块本身没有 `try_state` / try-runtime 逻辑，因此保持现状可接受。
- `shengbank-interest` try-runtime 传播：已正确传播。crate feature 包含 `frame-support/try-runtime`、`frame-system/try-runtime`、`sp-runtime/try-runtime`，并且 `lib.rs` 有 `try_state`。
- `resolution-issuance::migration` 的 `AllowedRecipients::get().is_empty()` 兜底：存在冗余路径。代码注释明确 fresh genesis、不做线上迁移；`genesis_build` 已写入 `AllowedRecipients`，`#[pallet::storage_version(migration::STORAGE_VERSION)]` 为 1。正常 fresh genesis 下 `on_chain >= STORAGE_VERSION` 会直接返回，兜底分支不会承担真实迁移职责。
- `shengbank-interest` 利率乘法溢出事件缺口：存在。代码先用 `bank.stake_amount.saturated_into()` 并通过 `ShengBankPrincipalOverflow` 审计本金转换截断，但随后 `principal.saturating_mul(rate_bp.into()) / 10_000` 没有 `checked_mul` 或 `ShengBankInterestOverflow` 事件。当前 runtime `Balance = u128` 且 `CHINA_CH` 的 `stake_amount` 远低于触发阈值，属于防御性审计缺口，不是当前参数下的可触发资金漏洞。
- 本轮未改业务代码，未运行测试。
