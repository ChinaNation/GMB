## 主要问题

1. `home/process::spawn_node` 以 `--unsafe-rpc-external --rpc-methods Unsafe --rpc-cors all` 启动节点，把本应本机自用的高权限 RPC 暴露到外部网卡。
2. `trusted_node_process_pids_on_rpc_port` 在兜底分支里会把“只要监听同一 RPC 端口且 RPC 指纹看起来像目标链”的进程当成可信节点，`cleanup_on_startup/exit` 可能误杀并非 nodeui 托管的本地链进程。
3. `set_reward_wallet` 仍然是“先本地保存，链上后台异步绑定”，调用方先拿到成功，后续链上失败只能靠事件补发；与 fee-address 技术文档声明的返回语义不一致。
4. 治理页 `开发升级` 标签和后端构建/提交命令始终暴露，没有在 nodeui 侧预检 `DeveloperUpgradeEnabled` 或管理员身份；运行时最终会拦，但前端展示和功能闭环没有严格实现“仅开发期可用”。
5. 文档和残留不干净：`memory/05-modules/citizenchain/nodeui/` 下没有 governance 技术文档；`cargo check/test` 持续报 `build.rs` 的 `unused_mut`、`governance/signing.rs` 的 dead code warning；`cargo test -p nodeui-desktop-shell` 还存在 3 个 `network-overview` 失败用例。
