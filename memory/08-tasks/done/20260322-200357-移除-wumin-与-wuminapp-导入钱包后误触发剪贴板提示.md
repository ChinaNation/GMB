# 移除 wumin 与 wuminapp 导入钱包后误触发剪贴板提示

## 任务背景

用户反馈 `wumin` 冷钱包与 `wuminapp` 在“导入钱包”成功后，会出现“已复制到剪切板/剪贴板”的系统提示。

当前导入流程为手动输入助记词，不提供粘贴入口，因此导入成功后主动写入系统剪贴板会造成误导。

## 当前状态

- `wumin/lib/ui/import_wallet_page.dart` 在导入成功后会执行一次 `Clipboard.setData('')`
- `wuminapp/lib/wallet/ui/wallet_page.dart` 的导入页在导入成功后也会执行一次 `Clipboard.setData('')`
- 两处行为的初衷是“清空剪贴板防止助记词残留”，但和当前手输导入流程不匹配

## 风险点

- 用户会误以为 App 将敏感信息复制到了系统剪贴板
- 冷钱包场景下会放大安全疑虑
- 若只改单端，会造成冷热钱包交互体验不一致

## 执行计划

1. 删除 `wumin` 导入成功后的剪贴板清空逻辑
2. 删除 `wuminapp` 导入成功后的剪贴板清空逻辑
3. 清理不再使用的 `flutter/services.dart` 依赖
4. 运行最小分析验证，确认无残留引用

## 实际处理

- 删除 `wumin/lib/ui/import_wallet_page.dart` 中导入成功后的 `Clipboard.setData('')`
- 删除 `wuminapp/lib/wallet/ui/wallet_page.dart` 中导入成功后的 `Clipboard.setData('')`
- 清理 `wumin/lib/ui/import_wallet_page.dart` 不再使用的 `flutter/services.dart` 引用

## 验证结果

- 通过 `rg` 确认两处导入页已不再包含“导入成功后清空剪贴板”逻辑
- `dart analyze` 已执行
- 当前剩余提示：
  - `wuminapp/lib/wallet/ui/wallet_page.dart:683`
  - `wuminapp/lib/wallet/ui/wallet_page.dart:1161`
- 以上 2 条均为既有 `use_build_context_synchronously` 提示，与本次修复无直接关系
