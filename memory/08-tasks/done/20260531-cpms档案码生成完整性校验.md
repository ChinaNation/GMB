任务需求：
CPMS 公民档案更新档案码时必须满足完整档案条件；同时选举资格必须由出生日期年龄约束，未满 16 周岁不得设置为有选举资格。

所属模块：
CPMS 后端档案业务、CPMS 前端档案创建/编辑/详情、CPMS 技术文档。

实现范围：
- 后端创建/编辑/状态更新档案时校验选举资格：公民状态正常且年满 16 周岁才允许 `voting_eligible=true`。
- 后端生成/打印 ARCHIVE 档案码前校验档案完整性：姓氏、名字、性别、身高、出生日期、护照号、有效期、省份、城市、公民状态、选举资格、投票账户、照片、出生纸齐全。
- 前端创建/编辑档案时按出生日期控制选举资格；未满 16 周岁时不可选择“有选举资格”。
- 前端更新档案码按钮在资料不完整时给出明确原因。
- 更新文档并清理散落残留。

验收标准：
- 未满 16 周岁的档案无法保存为有选举资格。
- 缺照片或出生纸时无法生成/打印档案码。
- 状态不是正常、选举资格不是有、投票账户缺失或必填身份字段缺失时无法生成/打印档案码。
- 前端构建、后端检查和残留扫描通过。

完成结果：
- 后端创建、编辑、状态更新、年度导出统一按公民状态和 16 周岁年龄线计算选举资格。
- 后端生成和打印 ARCHIVE 前新增完整性校验，缺身份字段、投票账户、照片或出生纸时拒绝签发。
- 前端创建/编辑按出生日期禁用“有选举资格”，详情页更新/打印档案码前展示中文缺失原因。
- 编辑实名字段、修改公民状态、绑定投票账户、上传资料和删除资料后统一清空旧档案码，避免继续展示过期 ARCHIVE。
- 年度报告重复导出时从当前档案数据重新生成，并覆盖同年度已记录报告。
- CPMS 技术说明、ARCHIVE 投票账户文档和 dangan 模块文档已同步更新。

验证：
- `cargo test --manifest-path cpms/backend/Cargo.toml`
- `npm run build`（`cpms/frontend`）
- `git diff --check`
- 残留扫描：旧选举资格函数、旧钱包判断函数、旧自动重签注释和旧签出描述无命中。
