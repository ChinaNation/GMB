# 任务卡：最后一次开发者直升——从开发期切换到运行期

- 任务编号：20260325-switch-to-production
- 状态：open（等 chain-phase-control 模块合入并在开发链验证后启动）
- 所属模块：citizenchain-runtime（跨多个 pallet 的一次性迁移）
- 当前负责人：待分配
- 创建时间：2026-03-25

## 背景

chain-phase-control 模块已实现，链上存储了三个关键参数：
- `Phase`（Development / Production）
- `TargetBlockTimeMs`（30,000 / 360,000）
- `DeveloperUpgradeEnabled`（true / false）

开发期通过 `developer_direct_upgrade` extrinsic 由 NRC admin 直接 set_code。
运行期通过 `propose_runtime_upgrade` 走联合投票升级。

本任务是**最后一次使用 developer_direct_upgrade 的机会**——执行后该调用永久失效。

## 开发期 vs 运行期对比

| 参数 | 开发期（当前） | 运行期（切换后） |
|------|---------------|-----------------|
| 链阶段 | Development | Production |
| 出块目标时间 | 30,000 ms（30 秒） | 360,000 ms（6 分钟） |
| 开发者直升 | 启用（NRC admin 直接 set_code） | 禁用（必须走联合投票） |
| GRANDPA 权威数 | 1 个（NRC 单节点） | 44 个（1 国储会 + 43 省储会） |
| grandpa-key-gov 映射 | 仅第 1 个 institution | 全部 44 个 institution |
| 难度调整窗口 | 基于 30 秒目标 | 基于 6 分钟目标（窗口重置） |
| runtime 升级方式 | developer_direct_upgrade | propose_runtime_upgrade → 联合投票 |

## 最后一次升级要做的事情

### 编写 SwitchToProduction 迁移

在 runtime 中编写一次性 `OnRuntimeUpgrade` 迁移，内容如下：

```rust
pub struct SwitchToProduction;

impl OnRuntimeUpgrade for SwitchToProduction {
    fn on_runtime_upgrade() -> Weight {
        // ① 关闭开发者直升——此后 developer_direct_upgrade 永久失效
        chain_phase_control::DeveloperUpgradeEnabled::<Runtime>::put(false);

        // ② 出块时间 30 秒 → 6 分钟
        chain_phase_control::TargetBlockTimeMs::<Runtime>::put(360_000u64);

        // ③ 标记链阶段为 Production
        chain_phase_control::Phase::<Runtime>::put(ChainPhase::Production);

        // ④ GRANDPA 从 1 个权威扩展到 44 个
        pallet_grandpa::Pallet::<Runtime>::schedule_change(
            full_44_authorities, Zero::zero()
        );

        // ⑤ 同步 grandpa-key-gov 映射（补全其余 43 个 institution→key 映射）
        for node in CHINA_CB.iter() {
            let inst = reserve_pallet_id_to_bytes(node.shenfen_id).unwrap();
            grandpa_key_gov::CurrentGrandpaKeys::<Runtime>::insert(inst, node.grandpa_key);
            grandpa_key_gov::GrandpaKeyOwnerByKey::<Runtime>::insert(node.grandpa_key, inst);
        }

        // ⑥ 重置难度窗口起点（避免目标时间 12 倍突变导致难度异常跳变）
        pow_difficulty_module::WindowStartMs::<Runtime>::kill();

        Weight::from_parts(500_000_000, 0)
            .saturating_add(RocksDbWeight::get().reads_writes(50, 100))
    }
}
```

### 挂载迁移到 Executive

```rust
pub type Executive = frame_executive::Executive<
    Runtime, Block, ..., AllPalletsWithSystem,
    SwitchToProduction,  // ← 一次性迁移
>;
```

### 执行步骤

1. 编译包含 SwitchToProduction 迁移的新 runtime WASM
2. 用 try-runtime 模拟迁移，确认无 panic
3. NRC admin 调用 `developer_direct_upgrade(new_wasm_blob)`
4. 新 runtime 在下一个区块首次加载时执行 SwitchToProduction
5. 确认 6 个写入全部生效
6. 确认 developer_direct_upgrade 已失效（调用应返回 DeveloperUpgradeDisabled）
7. 所有 44 个权威节点重启，加载各自 GRANDPA 密钥

### 执行后清理

下一个版本的 runtime 升级（走联合投票）中：
- 从 Executive 移除 SwitchToProduction 迁移（改回空 tuple `()`）
- 删除 SwitchToProduction 迁移代码

## 风险与对策

| 风险 | 对策 |
|------|------|
| 迁移执行失败导致链卡住 | 先用 try-runtime 模拟通过再上链 |
| GRANDPA schedule_change 延迟 | delay=0 下一个 finalized block 后生效；过渡期 1 个权威签名不影响安全 |
| 出块时间突变导致难度异常 | 迁移中 kill() 难度窗口起点，强制重新计算 |
| 节点未重启导致门控时间不对 | 节点启动时从 Runtime API 读一次目标时间；切换后需重启节点 |
| 忘记清理迁移代码 | 下一版本 CI 检查 Executive 的迁移参数不得包含已执行的 struct |
