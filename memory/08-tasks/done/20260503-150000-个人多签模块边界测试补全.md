# 个人多签模块边界测试补全

## 背景

2026-05-03 全仓审计发现 [duoqian-manage::personal](citizenchain/runtime/transaction/duoqian-manage) 业务路径完整闭环(95% 完成度),但单测仅覆盖正常路径 + 几类异常路径,边界 case 缺 8 个。补齐后达 100% 完成度,可正式发布。

## 边界 case 清单(8 个)

| # | 用例 | 期望 |
|---|---|---|
| 1 | `account_name` 长度 = `MaxAccountNameLength`(128)边界:刚好 128 字节通过,129 字节被 BoundedVec 拒 | encode 失败前/后两路径 |
| 2 | `admins.len()` = 1 拒(`InvalidAdminCount`,`MaxAdmins=10` 但需要 ≥2)+ `admins.len()` = `MaxAdmins=10` 通过 | 上下界 |
| 3 | `threshold = 0` 拒 / `threshold > admins.len()` 拒 / `threshold = admins.len()` 通过 | threshold 三态 |
| 4 | `amount = MinCreateAmount-1`(110)拒 + `amount = MinCreateAmount`(111)通过 | 资金最低门槛 |
| 5 | 同 creator + 同 account_name → 派生地址相同 → `PersonalDuoqianAlreadyExists` | 重复创建保护 |
| 6 | propose_create 后第二个 propose_create 同名(DuoqianAccounts Pending)→ 同 5 路径触发 | 并发 propose 防御 |
| 7 | 投票否决后 `cleanup_rejected_proposal`:reserve 已 unreserve / 4 张 storage 全清 / `DuoqianCreateRejected` 事件 | 否决路径 storage 残留检查 |
| 8 | beneficiary = duoqian_address 自身 / 保护源 → propose_close 拒 | 恶意收款人 |
