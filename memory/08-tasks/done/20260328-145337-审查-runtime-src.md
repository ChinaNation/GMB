## 主要问题

1. `PowDigestAuthor` 仍然只信任 pre-runtime digest 中可解码的 sr25519 公钥，奖励与手续费分成继续继承“作者身份可伪造”的协议边界问题。
2. `RuntimeJointVoteResultCallback` 的联合投票回调路由仍然偏脆弱：只对 `resolution_issuance_gov` 先做 ownership 校验，其他情况直接尝试 `runtime_upgrade` 回调，没有统一模块标签/归属判断。
3. `PowTxAmountExtractor` 对 `Balances::transfer_all` 仍按扣费前的 `reducible_balance` 计费，实际执行时转账金额会因为手续费先扣而变小，收费口径偏大。
4. `offchain-transaction-pos` crate 与技术文档都存在，但 `runtime` 根本没接线：`runtime/Cargo.toml` 没依赖，`construct_runtime!` 没注册，`configs/mod.rs` 没 `Config` 实现，属于明确功能缺口。
5. 文档和测试残留未清理：`memory/05-modules/citizenchain/runtime/README.md` 只有目录说明；`cargo test -p citizenchain` 还有 3 个红测，其中 `genesis_config_presets` 两个断言已经和当前创世实现漂移，`configs` 里还有一个联合投票回调测试直接按旧数据格式解码失败。
