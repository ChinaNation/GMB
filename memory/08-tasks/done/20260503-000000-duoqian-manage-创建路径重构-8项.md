# duoqian-manage 创建路径重构（8 项 + wumin 同步）

## 背景

承接前一轮拆分（20260502-230000）后,duoqian-manage 创建路径暴露 8 个改进/缺口:

1. 单账户机构 `propose_create` (call=0) 是历史包袱,业务上只允许"机构最少 2 账户"
2. 个人多签未用 reserve 资金模型,与机构多签不一致(投票期间发起人资金可被花掉,执行时可能失败)
3. 机构治理索引绕道 main_address,语义应直接用 sfid_id(= shenfen_id)
4. 创建提案当前用用户 m-of-n 阈值通过,**应该是全员通过**(主体生命周期类语义)
5. 关闭提案当前用 m-of-n 阈值,**应该是全员通过**(同上)
6. ACTION_CREATE 删 call=0 后语义只剩个人多签,改名 ACTION_CREATE_PERSONAL
7. 个人/机构创建路径有重复参数校验,可抽 ensure_admin_config 共用
8. 余额预检查代码重复,可抽 helper

## 边界

- spec_version 不动(用户拍板)
- chainspec.json 不改
- ABI: 删 call_index=0 是破坏性变更,但当前 fresh genesis 链上无数据
- 不动管理员更换(归 admins-change,本任务不讨论)
- 不动测试模块拆分(用户已明确)
- 链上 0 数据(无 sfid_id 已派生,改索引算法无迁移成本)

## 业务规则

| 提案类型 | 投票阈值规则 |
|---|---|
| 创建多签(个人/机构) | **全员**(threshold = admins.len()) |
| 关闭多签 | **全员**(threshold = admins.len()) |
| 转账 | m-of-n(subject.threshold) |
| 管理员更换 | m-of-n(subject.threshold,本任务不动) |

`admins-change::Institutions[id].threshold` 字段保存用户自定义 m-of-n,用于激活后日常治理(转账等),与"创建/关闭全员通过"不冲突。

## 改动清单（按 Step）

### Step A: 删除 propose_create call=0
- `duoqian-manage::lib.rs`: 删除 `propose_create` extrinsic（call_index=0 留洞,绝不复用）
- `institution/create.rs`: 删除 `do_propose_create` 函数
- `weights.rs`: 删除 `propose_create()` weight
- `tests`: 删除 propose_create 相关用例,迁移到 propose_create_institution（如需要）

### Step B: voting-engine 加新入口
- `voting-engine::lib.rs`: `InternalVoteEngine` trait 新增方法
  ```rust
  fn create_internal_proposal_with_threshold_and_data(
      who: AccountId,
      org: u8,
      institution: InstitutionPalletId,
      threshold: u32,
      module_tag: &[u8],
      data: Vec<u8>,
  ) -> Result<u64, DispatchError>;
  ```
- 内部实现：admins 仍从 active 主体反查,threshold 显式传入,不调 `pass_threshold`

### Step C: sfid_id 治理索引正名
- `duoqian-manage::common.rs`: 新增 `sfid_id_to_institution_id(sfid_id)` 调用 `primitives::china::china_cb::shenfen_id_to_fixed48`
- `do_propose_create_institution`: `let institution = sfid_id_to_institution_id(&sfid_id)?` 替代 `account_to_institution_id(&main_address)`
- `resolve_admin_subject_for_account`: 反查路径改 `AddressRegisteredSfid → sfid_id → sfid_id_to_institution_id`,不再绕 `Institutions[sfid_id].main_address`
- `institution/execute.rs` 同步

### Step D: 个人多签 reserve 资金模型
- 新增 storage `PendingPersonalCreate[proposal_id] = CreateDuoqianAction`(与机构 PendingInstitutionCreate 对应)
- `do_propose_create_personal`: 创建时立即 `reserve(who, amount + fee)` + 写 `PendingPersonalCreate`
- `execute.rs::execute_create_with_finalizer`: `unreserve + withdraw fee + transfer`(个人多签分支)
- callback 否决/失败终态: 加 unreserve + 删 PendingPersonalCreate

### Step E: 创建提案改全员投票
- `do_propose_create_personal`: 传给 voting-engine 的 threshold 改 `duoqian_admins.len() as u32`
- `do_propose_create_institution`: 同上

### Step F: 关闭提案改全员投票
- `do_propose_close`: 改调 Step B 新入口 `create_internal_proposal_with_threshold_and_data`,传 `admins.len()`
- 注意: admins 数量从 active 主体读(`admins-change::active_subject_admin_count`)

### Step G: ACTION_CREATE 改名
- `pallet::ACTION_CREATE` → `ACTION_CREATE_PERSONAL`(数值仍 = 1,SCALE 编码不变,decoder 字面识别能力不变)
- `personal/create.rs` / `execute.rs` / `lib.rs::InternalVoteExecutor` 同步
- `wumin::action_labels.dart`: 删 `propose_create` 中文标签(已废弃)；保留 `propose_create_personal` 标签
- `wumin::pallet_registry.dart`: 删 `proposeCreateCall = 0` 常量,call_index 留洞
- `wumin::payload_decoder.dart`: 删 `_decodeProposeCreate` 分支
- `wumin::test/signer/payload_decoder_test.dart`: 删测试
- `citizenchain/node/governance/signing.rs`: 如有 build_propose_create 同步删除

### Step H: ensure_admin_config 共用
- `do_propose_create_personal` 改调 `Pallet::<T>::ensure_admin_config(&who, admin_count, &duoqian_admins, threshold)`,删除内联校验

### Step I: 余额预检查 helper
- `common.rs` 新增 `ensure_proposer_can_afford<T>(who: &AccountId, amount: BalanceOf, fee: BalanceOf) -> Result<(), Error>`
- 个人多签 / 机构多签 / 单账户机构（待删）调用 helper

## 执行顺序

A → B → C → D → E → F → G → H → I → wumin 同步 → 验收

每步 cargo check 通过才进下一步;Step C/D/E/F 后跑 cargo test;最后跑 flutter analyze + flutter test。
