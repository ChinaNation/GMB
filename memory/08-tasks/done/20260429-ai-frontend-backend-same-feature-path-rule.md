# 任务卡：前后端同功能同目录同名规则

## 任务需求

将用户确认的前后端目录规则写入 GMB AI 编程系统强制规则：

- 相同的功能应该在前后端创建相同的文件夹。
- 如果功能不大，则直接在文件夹下面创建相同的文件。
- 如果功能太大，则视情况再建一级文件夹。
- 如果不确定，则先询问用户。

同时基于该规则，输出清算行在 `node/offchain` 下的前后端目录方案。

## 建议模块

- `memory/07-ai`
- `memory/08-tasks`
- `citizenchain/node`

## 影响范围

- AI 编程系统强制规则
- AI 开发流程门禁
- 后续前后端功能目录设计
- 清算行 node/offchain 改造方案

## 技术方案

1. 更新 AI 编程系统规则
   - 在 `memory/07-ai/agent-rules.md` 增加前后端同功能同目录同名规则。
   - 在 `memory/07-ai/workflow.md` 增加同样的执行门禁。

2. 输出清算行 node/offchain 目录方案
   - 后端统一收口到 `citizenchain/node/src/offchain`。
   - 前端统一建立 `citizenchain/node/frontend/offchain`。
   - 前后端按同名文件或同名子文件夹一一对应。

3. 文档、注释、清理残留
   - 本任务只更新 AI 系统规则文档，不改业务代码。
   - 不新增无意义注释。
   - 不创建临时文件或兼容目录。
