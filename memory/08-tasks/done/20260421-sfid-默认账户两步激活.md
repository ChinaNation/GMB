# 任务卡:SFID 所有机构默认账户 + 两步激活模式

- 任务编号:20260421-sfid-default-accounts-two-step-activate
- 状态:done
- 完成日期:2026-04-21
- 负责人:主聊天入口(SFID Agent)

## 1. 目标

把上一轮(2026-04-19 第四轮)的"清算行设置"概念彻底废除,**把主账户 / 费用账户的机制扩展到所有机构**:

- 公权(GFR) / 私权(SFR/FFR) / 公安局(PublicSecurity)新建 SFID 时,**后端自动**在账户列表插入 `主账户`、`费用账户`两条记录
- 这两条记录初始状态是 `Inactive`(不上链),由 sfid 管理员显式"激活"才推链
- 管理员手工建的其他账户**统一走同一两步模式**(Inactive → 显式激活)
- 默认账户不可删除,其他账户可软删(不触链)

## 4b. 同日追加:本地地址派生 + 公安局分片同步 bug 修复

### 新增 `institutions/derive.rs`

DUOQIAN_V1 本地派生工具,与链端 `Pallet::derive_institution_address` 字节对齐:
```
preimage = b"DUOQIAN_V1" (10B)
        || op_tag (1B: 0x00 主账户 / 0x01 费用账户 / 0x05 自定义)
        || 2027_u16.to_le_bytes() (2B ss58 prefix)
        || sfid_id.as_bytes()
        || account_name.as_bytes()  // 仅 0x05 追加
duoqian_address = blake2b_256(preimage) (32B)
```

`derive_duoqian_address(sfid_id, account_name) -> Option<String>` — 按 account_name 字节路由(`"主账户"` / `"费用账户"` / 其他),返回 hex(64 字符 小写)。

### 3 处立即填 duoqian_address

- `handler::insert_default_accounts_best_effort`(create_institution 路径)
- `service::insert_default_accounts_into_legacy`(公安局 reconcile 路径)
- `handler::create_account`(管理员手工建)

创建时 `MultisigAccount.duoqian_address = Some(derive(...))`,前端立即可见,无需等激活上链。

### 激活一致性断言

`activate_account` 收到链上 receipt 后:`expected = derive_duoqian_address(sfid, account_name)`;与 `receipt.duoqian_address` 不等 → 500 "地址派生不一致"。抓 domain / op_tag / ss58 漂移。

### 公安局 reconcile sharded_store bug 修复

**Bug**:reconcile 是同步函数只写 legacy store,sharded_store 没同步 → 详情页按省读分片查不到账户列表。

**修复**:新增 `app_core::runtime_ops::sync_public_security_to_sharded(state)`,main.rs runtime 起来 + preload_all_shards 完成后调用。幂等:
1. 遍历 legacy 所有 `PublicSecurity` 机构 + 其已有账户 → 按省分组写入 sharded_store(`or_insert` 保留已激活条目)
2. 对缺失默认账户的老公安局(历史遗留),走 `derive_duoqian_address` 补齐,legacy + sharded 双写

### 测试

新增 `institutions::derive` 5 个单元测试:
- `main_account_preimage_excludes_name` — 不同 sfid 的"主账户"地址不同

- `named_uses_account_name_in_preimage` — 自定义账户用 account_name;英文 "Main" 作为 Named 与"主账户"不等
