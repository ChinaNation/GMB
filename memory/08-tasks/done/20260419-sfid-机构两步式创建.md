## 3. 关键变更

1. **两步式私权创建**
   - 第一步 `POST /institution/create`:只生 SFID,`institution_name`/`sub_type` 不接受
   - 第二步 `PATCH /institution/:sfid_id`:详情页保存 name + sub_type
   - 创建成功后立即跳详情页(`InstitutionsView.onCreated` 判断 PRIVATE_INSTITUTION 时 `setSelectedSfidId`)

2. **sub_type↔P1 联动 + NON_PROFIT**
   - 新增 `service::validate_sub_type_with_p1`
   - `VALID_SUB_TYPES` 增加 `NON_PROFIT`
   - `P1=0`→强制 NON_PROFIT;`P1=1`→禁止 NON_PROFIT

3. **机构代码清理(死规则,不保留兼容)**
   - GFR 选项移除 `CB`(储备委员会)
   - SFR/FFR 选项移除 `CH`(储备银行)及 `institutionChoicesForSubType` / `SFR_INSTITUTIONS_ALL` 等分叉逻辑

4. **数据模型变更**
   - `MultisigInstitution.institution_name: String` → `Option<String>`
   - `CreateInstitutionInput.institution_name: String` → `Option<String>`
   - 新增 `UpdateInstitutionInput { institution_name, sub_type }`
   - JSON 持久化通过 `#[serde(default, skip_serializing_if=Option::is_none)]` 兼容老记录

5. **名称唯一性函数**
   - 新增 `service::institution_name_exists_excluding(store, name, exclude_sfid_id)`,用于 update 时排除自身
   - `institution_name_exists` / `institution_name_exists_in_city` 对齐 `Option<String>` 比较

6. **UI 交互**
   - 详情页新增"完善机构信息" Card(机构名称 + 企业类型)
   - 未命名机构:"+ 新建账户" 按钮禁用 + Alert 提示"先完善机构名称与企业类型"
   - 机构列表 `institution_name=null` 显示占位"(未命名,待完善)"
