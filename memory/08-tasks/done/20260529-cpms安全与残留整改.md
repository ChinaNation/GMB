# 任务卡：CPMS安全与残留整改

## 任务需求

按已确认方案整改 CPMS 系统安全、流程和残留问题：安装初始化只做本地防误装校验，不加密安装码，不新增二维码；生产密钥不得明文落库；初始化必须事务完整；操作管理员只允许新增和删除；档案删除签名流程加事务锁；QR 登录唯一化并改为 HttpOnly Cookie；清理旧登录、禁用、兼容、过渡和占位残留；更新文档并完善中文注释。

## 建议模块

- CPMS 后端
- CPMS 前端
- CPMS 部署脚本
- CPMS 数据库 schema/migration
- CPMS/CID 相关技术文档

## 影响范围

- `citizenpassport/backend`：初始化、鉴权、登录、管理员、档案、地址校验、编号/签发相关代码。
- `citizenpassport/backend/db`：当前基准 schema 与迁移残留。
- `citizenpassport/deploy`：生产安装脚本和环境变量。
- `citizenpassport/frontend/web/src`：登录态、管理员列表、档案详情和 API。
- `memory/05-modules/citizenpassport`：模块文档、错误码和流程说明。
- `memory/05-modules/citizencode`：两码模型边界说明。

## 主要风险点

- Cookie 登录态改造需要前后端一致，不能造成登录后接口 401。
- 操作管理员物理删除后必须保留审计快照，不能让审计断线。
- 初始化事务收口不能留下半初始化状态。
- 删除旧迁移和旧登录接口时不得保留兼容分支。
- 编号回收和 100 年物理删除按用户要求另建任务，不在本次实现。
