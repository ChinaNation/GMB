# wuminapp 个人多签详情页 UI 改造(req 1/2/4/5)

## 背景

wuminapp 个人多签创建后的详情页 UI 不完善:管理员列表平铺浪费空间、待激活状态下转账按钮可点(逻辑错误)、非创建者管理员无独立"激活/签名"入口、详情页看不到该多签的提案列表。本任务卡就 4 项需求做一次性整改(req 3 用户决策延后,本卡不实现)。

## 用户决策记录

- **req 3(钱包账户反查多签)**:**本期不做**(链上无反向索引,实现复杂度高)
- **req 5 历史提案策略**:链上活跃提案(链上 90 天后清理终态提案,已 verified [voting-engine/lib.rs:23](citizenchain/runtime/governance/voting-engine/src/lib.rs:23))→ wuminapp 自己 Isar 全量记录历史
- **req 2 激活语义**:创建人发起 = 已同意,无需自己再激活;"激活" = 其他管理员对当前创建提案投赞成票(complex 当前 internal_vote approve 路径)
- **req 4 禁用样式**:沿用现有 [proposal_types_page.dart:404](wuminapp/lib/citizen/proposal/proposal_types_page.dart:404) 的"显示但灰色不可点"模式(`enabled=false` + `Opacity 0.5` + `onTap: null`)

## 4 项需求详解

### req 1 — 管理员列表折叠成单行,点进子页

**现状:** [duoqian_account_info_page.dart:392-453](wuminapp/lib/duoqian/shared/duoqian_account_info_page.dart) 平铺所有管理员 ListTile。

**改造:**
- 详情页该卡片改为单行 `ListTile`,文案 "管理员列表(N 人)" + 右箭头
- 点击 → 新页 `PersonalAdminListPage`(参考 [admin_list_page.dart](wuminapp/lib/citizen/institution/admin_list_page.dart) 的机构管理员页结构)

### req 2 — 子页非创建者管理员显示"激活"按钮

**触发条件:** 仅当 `info.status == DuoqianStatus.pending` 时显示;创建者那行**不显示**(创建即同意);
**按钮文案三态:**
| 提案状态 + 该 admin 投票状态 | 按钮文案 + 行为 |
|---|---|
| 投票中(STATUS_VOTING)+ 该 admin 未投 | "激活"(可点击,蓝色) |
| 投票中 + 该 admin 已投赞成 | "已激活"(灰色禁用) |
| 投票中 + 该 admin 已投反对 | "已拒绝"(灰色禁用,不可改投,沿用 voting-engine 单次投票规则) |
| 创建提案已 REJECTED / 已 EXECUTED | 整页跳转/无关:多签若激活成功,info.status 已变 Active,本页不再显示激活按钮;若被否决,DuoqianAccount 已删除,UI 应跳回列表 |

**点击行为:** `InternalVoteService.submitInternalVote(proposal_id, approve=true)`(已存在,[internal_vote_service.dart:41](wuminapp/lib/citizen/proposal/shared/internal_vote_service.dart:41))

**关键依赖:** 当前 admin 子页要拿到 `proposal_id`。来源 = `PendingPersonalCreate::iter()` 找匹配 personal_address 的项,或从详情页透传(若详情页本身不知道,则 service 加 lookup 方法)。

### req 4 — 待激活时转账按钮置灰

**现状:** [duoqian_account_info_page.dart:388](wuminapp/lib/duoqian/shared/duoqian_account_info_page.dart:388) `_buildTransferEntryCard()` 永远可点击。

**改造:** 检测 `info.status != DuoqianStatus.active` → `enabled = false`(灰色 + Opacity 0.5 + `onTap: null`),副标题"账户尚未激活,无法发起转账"

### req 5 — 详情页底部加"提案列表"

**数据源 — 双轨制:**
1. **活跃提案(STATUS_VOTING)** → 链上 `voting_engine.activeProposalsByInstitution(blake2_128(personal_address))` 实时拉,展示完整投票进度
2. **历史提案(REJECTED / EXECUTED / EXECUTION_FAILED)** → wuminapp 本地 Isar 全量缓存(链上 90 天会清,本地永不清)

**展示形式:** 详情页底部一个 Card,标题 "该多签提案",列表项点击进入 [duoqian_manage_detail_page.dart](wuminapp/lib/duoqian/shared/duoqian_manage_detail_page.dart) 沿用现有提案详情。

**Isar 模型新增:** `PersonalDuoqianProposalEntity { personalAddress, proposalId, action(create/close/transfer), status, voteTally, createdAtMs, finalStatusAtMs, snapshot }`,在以下时机写入:
- 用户在本设备发起 propose_create_personal / propose_transfer / propose_close 后
- 用户在本设备给某提案投票后(刷新该提案状态)
- 详情页打开时同步活跃提案最新状态写回(防止其他设备发起的提案漏记)

### 落地形式 vs 设计

实施完全对齐设计方案,但有 1 处务实简化:

- **req 2 激活按钮的点击行为**:设计方案是直接在 admin 子页内调 `InternalVoteService.submit(approve=true)` + QrSigner 签名流程。实施时改为**跳转到现有 [DuoqianManageDetailPage](wuminapp/lib/duoqian/shared/duoqian_manage_detail_page.dart)**(传入 proposalId + admin role context),让用户在那里完成投票。理由:detail page 已经包含完整的 Phase 3 投票签名流(QrSigner / PendingVoteStore / 错误处理),复用避免重复实现;UI 上"激活"按钮跳到投票页用户不会困惑(投赞成 = 激活,语义对齐)。投票完成后 admin 子页 setState 刷新,该行变"已激活"。

### 双轨制实际工作流(req 5)

链上活跃提案数据流(STATUS_VOTING):
- `_safeFetchActiveProposalIds` → 查 `voting_engine.ActiveProposalsByInstitution[personal_address || zeros(16)]`
- 个人多签 InstitutionPalletId = `personal_address(32B) || 16B 零右补`(对齐 [duoqian-manage::common::account_to_institution_id](citizenchain/runtime/transaction/duoqian-manage/src/common.rs:19))**无 hash**,本服务通过自定义 `_personalAddressToInstitutionId` 编码,与 `transfer_proposal_service::_institutionIdentityToFixed48`(SFID 路径)区分

历史提案数据流(REJECTED / EXECUTED / EXECUTION_FAILED):
- 链上 90 天清理(verified [voting-engine/lib.rs:23](citizenchain/runtime/governance/voting-engine/src/lib.rs:23) "90 天延迟分块清理"),Isar 永久保留
- 写入触发点 3 处:
  1. 创建提案提交 → [personal_duoqian_create_page.dart](wuminapp/lib/duoqian/personal/personal_duoqian_create_page.dart) 提前 query NextProposalId,提交后写 Isar
  2. 关闭提案提交 → [personal_duoqian_close_page.dart](wuminapp/lib/duoqian/personal/personal_duoqian_close_page.dart) 同上
  3. 转账提案提交 → [transfer_proposal_page.dart](wuminapp/lib/citizen/proposal/transfer/transfer_proposal_page.dart) `_maybeRecordPersonalProposal` 检查 Isar 是否有 `PersonalDuoqianEntity` 命中 → 命中视为个人多签 → 写 Isar 提案 entity
- 同步增量:`PersonalProposalHistoryService.fetchAll` 在详情页 mount 时调用,把链上 active 提案 upsert 到 Isar(防其他设备发起的提案漏记)
