## 已确认结论

- 本机节点当前运行正常但 `system_health.peers=0`，未建立任何 P2P 邻居。
- 本机节点与服务器节点使用同一条链，`chain_getBlockHash(0)` 均为 `0xea8fdcf52a9580381c5e8c38f91741b4fdc2ae787b5a646ffa19975cd056e4a9`。
- 本机运行中的 sync spec 已包含 44 条 `wss://30333` bootnodes，说明不是“本机启动时丢失 bootnodes”。
- 第二台服务器 `prczss.wuminapp.com` 当前在线，`PeerId=12D3KooWPjWNXvCzPv6PPuiGnF3J5uToW3ySfaB7rKkwUrN2CALv`，且日志显示 `Idle (2 peers)`。
- 第二台服务器的 `ss -tnp | grep 30333` 已确认它当前与第一台服务器 `147.224.14.117:30333` 保持已建立连接，说明两台服务器节点并未整体断网。
- 本机节点当前只对两台服务器的 `9944` 发起 UI 侧网络概览探测（`SYN_SENT`），未观察到稳定的 `30333` 出站连接，问题集中在“本机未成功加入现有 P2P 网络”。
- 本机节点版本为 `1.0.0-19fc5b02bd5`，第二台服务器节点版本为 `1.0.0-7492b298092`，当前本机与服务器运行版本不一致。
