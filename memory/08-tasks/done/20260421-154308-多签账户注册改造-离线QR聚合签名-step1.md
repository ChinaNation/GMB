- **新增** runtime 层 `DuoqianInternalAdminProvider` / `DuoqianInternalThresholdProvider` 实现(从 `DuoqianAccounts` 读,允许 Pending 返回快照)
- **投票引擎**:0 改动

## 技术方案(与上轮对话定稿一致)

### 1. 常量(已就位)

```
DUOQIAN_DOMAIN = b"DUOQIAN_V1"  (primitives::core_const,10 B)
OP_SIGN_CREATE = 0x14           (primitives::core_const,1 B)
```

### 2. 新结构 `CreateVoteIntent`

```rust
#[derive(Encode, Decode, TypeInfo)]
pub struct CreateVoteIntent<AccountId, Balance> {
    pub proposal_id: u64,
    pub duoqian_address: AccountId,
    pub creator: AccountId,
    pub admins_root: [u8; 32],        // blake2_256(SCALE.encode(sorted_admins))
    pub threshold: u32,
    pub amount: Balance,
    pub approve: bool,                // 恒 true,占位防误签
}
```

### 3. 签名消息

```
preimage =
    DUOQIAN_DOMAIN (10B)
 || OP_SIGN_CREATE (1B)
 || SS58_PREFIX_LE (2B = 2027_u16.to_le_bytes())
 || blake2_256(intent.encode())
signed_message = blake2_256(preimage)   // 32B
signature      = sr25519_sign(admin_key, signed_message)
```

### 4. `finalize_create(proposal_id, sigs)` 流程

1. 读 ProposalData (MODULE_TAG + ACTION_CREATE + action)
2. 读 DuoqianAccount(Pending)并验 `creator == action.proposer`
3. 重算 `admins_root`、构造 intent、拼 signed_message
4. 循环 (admin, sig):成员校验 + 去重 + `sr25519_verify` + `cast_internal_vote(admin, proposal_id, true)`
5. 投票引擎内部自动阈值检查,达标自动 STATUS_PASSED
6. 读 status:PASSED → `execute_create`;REJECTED → 清 Pending

### 5. Runtime Provider

- `DuoqianInternalAdminProvider::{is_internal_admin, get_admin_list}` 从 `DuoqianAccounts` 读(不过滤 status)
- `DuoqianInternalThresholdProvider::pass_threshold` 同源

### 6. 新增错误码

```
UnauthorizedSignature / DuplicateSignature / InvalidSignature
InsufficientSignatures / MalformedSignature
```

### 7. 新增事件

```
CreateFinalized { proposal_id, signatures_accepted, final_status }
DuoqianCreateRejected { proposal_id, duoqian_address }
```

## 风险

- 投票引擎管理员快照读 `DuoqianAccounts` 时必须不过滤 status(Pending 也要返回)→ runtime Provider 实现中强制不加 status 过滤
- `admin_count` 上限需保证 `sigs` 数组编码不超链上交易大小 → 用 `BoundedVec<_, MaxAdmins>`
- sr25519 签名长度固定 64 字节 → 新增 `MaxAdminSignatureLength = 64` 常量并 BoundedVec 约束
- 发起人不必是管理员(Tx 1 已锁了 `action.proposer`),Tx 2 仅代投 + 代付 gas
