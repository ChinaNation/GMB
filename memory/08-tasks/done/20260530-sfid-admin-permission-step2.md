# SFID 省市管理员权限分级第 2 步任务卡

## 任务目标

同时设计并落地省级管理员、市级管理员的权限分级:

- 省级管理员可进行本省所有操作。
- 市级管理员只能进行本市业务操作。
- 查看操作只需要登录态。
- 一般操作必须 Passkey。
- 重要操作必须 Passkey + 当前管理员冷钱包 sr25519 挑战签名。
- 全仓库只使用 `WUMIN_QR_V1`,不得新增二维码协议。

## 固定约束

- 不做旧流程兼容。
- 不保留省级/市级旧权限分支。
- 不保留旧代签流程。
- 改代码后必须更新文档、完善中文注释、清理残留。

## 范围

- 后端通用管理员安全动作分级。
- Passkey 从省管理员专用泛化为省/市管理员通用。
- CPMS 授权治理仍归省管理员,市管理员禁止。
- 公民、机构业务按省/市 scope 放行。
- 前端能力矩阵与安全动作弹窗接入。
