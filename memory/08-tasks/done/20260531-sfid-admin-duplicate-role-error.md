# SFID 管理员重复账号按角色返回错误

## 任务目标

新增省级管理员和市级管理员时，管理员公钥继续保持全局唯一，但重复账号错误必须区分“已是省级管理员”和“已是市级管理员”。

## 修改范围

- `sfid/backend/admins/`：统一管理员公钥查重逻辑，按已存在管理员角色返回明确冲突。
- `sfid/backend/main.rs`：补充稳定业务错误码映射。
- `sfid/frontend/admins/`：按稳定错误码显示中文提示，不解析后端英文文案。
- `memory/`：补充管理员公钥全局唯一和重复错误码说明。

## 验收标准

- 新增省级管理员时，重复省级管理员显示“该账户已是省级管理员，不能重复新增”。
- 新增省级管理员时，重复市级管理员显示“该账户已是市级管理员，不能新增为省级管理员”。
- 新增市级管理员时，也能区分已是省级管理员和已是市级管理员。
- 后端和前端检查通过，残留旧错误文案清理完成。

## 完成内容

- 后端新增统一管理员公钥查重 helper，新增省/市管理员时按已存在角色返回冲突。
- 后端新增 `SFID_ADMIN_PUBKEY_EXISTS_AS_SHENG_ADMIN` 与 `SFID_ADMIN_PUBKEY_EXISTS_AS_SHI_ADMIN` 稳定错误码。
- 前端新增管理员公钥重复错误码文案转换，新增省/市管理员失败时显示明确中文原因。
- 文档已补充管理员公钥全局唯一和角色级重复错误码规则。
- 已清理旧的省级/市级管理员重复账号英文笼统报错残留。
