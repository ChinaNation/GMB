1. 功能定位
fullnode桌面前端主要面向全节点，是全节点的前端操作软件，提供区块数据归档、PoW 铸块监控、节点状态管理、日志查看等功能：
	•	节点状态监控：区块高度、区块同步状态、网络连接状态、节点活跃度。
	•	PoW 铸块管理：查看当前挖矿状态、矿工奖励统计、挖矿线程管理。
	•	交易数据查看：链上交易总览、交易手续费分配情况。
	•	日志与告警：全节点运行日志、异常警告、系统通知。
	•	配置管理：链的 bootnodes、数据库路径、RPC 接口配置。
	•	升级和维护：节点程序升级提示、数据库压缩、备份等。

2. 目录结构设计
gmb/
└── frontend/                     # 所有节点前端统一放在这个文件夹
    └── fullnode/                 # 全节点前端
        ├── package.json          # 前端依赖管理文件
        ├── public/               # 公共资源文件（图片、图标、静态文件等）
        ├── node_modules/         # 前端依赖库（npm/yarn 安装的包）
        └── src/                  # 源代码文件夹
            ├── App.tsx           # 主入口文件
            ├── components/       # 可复用组件，例如区块列表、节点状态面板
            ├── pages/            # 页面模块，例如监控页、交易页、日志页
            └── utils/            # 工具函数，例如 RPC 调用、数据处理

3. 前端框架推荐
	•	Tauri：桌面端打包，轻量、开源、安全。
	•	React：前端组件化，便于维护和扩展。
	•	MUI (Material UI)：美观、开箱即用的 UI 库。

⸻
4. 前端与节点交互
	•	通过 RPC 或 WebSocket 与节点后台通信。
	•	调用 pallet 提供的接口，例如：
	•	查询区块状态 (fullnode-pow-reward)
	•	查询交易分账 (transaction-fee-distribution)
	•	查询链上治理状态 (governance)
	•	所有操作都只在桌面前端显示和管理，不直接修改区块链状态（状态变化由 runtime/pallet 决定）。
